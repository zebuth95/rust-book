# Reference Cycles

`Rc<T>` can create reference cycles that **leak memory** (count never reaches 0).

## Solution: `Weak<T>`

`Weak<T>` is a non-owning reference. It doesn't increment the reference count.

```rust
use std::rc::{Rc, Weak};
use std::cell::RefCell;

struct Node {
    value: i32,
    parent: RefCell<Weak<Node>>,
    children: RefCell<Vec<Rc<Node>>>,
}
```

Use `weak.upgrade()` to get an `Option<Rc<T>>` (returns `None` if the value was dropped).

> **Note:** Use `Weak<T>` for back-references to break cycles.
