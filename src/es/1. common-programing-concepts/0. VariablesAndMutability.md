# Variables y Mutabilidad

Las variables son **inmutables** por defecto.

## `let` (inmutable)

```rust
let x = 5;
// x = 6;  // ❌ compile error: cannot assign twice
```

## `let mut` (mutable)

```rust
let mut x = 5;
x = 6;  // ✅ allowed
```

## `const` (siempre inmutable)

```rust
const MAX_POINTS: u32 = 100_000;
```

## Shadowing

Re-declarar con `let` para crear una nueva variable (puede cambiar de tipo).

```rust
let x = 5;
let x = x + 1;       // shadows, x = 6
let x = "hello";     // shadows with different type
```

> **Punto clave:** Shadowing NO es mutación. El valor antiguo es reemplazado.

## `static` — Variables Globales

Una variable `static` tiene una **dirección de memoria fija** y vive durante todo el programa.

```rust
static MAX_CONNECTIONS: u32 = 1024;
static mut COUNTER: u32 = 0;  // unsafe to read/write
```

- Las variables `static` **no se insertan** en tiempo de compilación.
- Los estáticos mutables (`static mut`) son inherentemente inseguros y requieren bloques `unsafe` para leer o escribir.

## `const` vs `static`

| | `const` | `static` |
|---|---|---|
| Insertado en tiempo de compilación | Sí | No (dirección fija) |
| Tiene dirección de memoria | No | Sí |
| Mutación insegura | N/A | Permitido con `static mut` |

## Ámbito y Shadowing (En Profundidad)

El shadowing permite reutilizar un nombre de variable dentro de un ámbito al redeclararla con `let`. La nueva variable **oculta** la anterior.

### Shadowing en ámbitos anidados

```rust
let x = 5;
{
    let x = 99;       // shadows outer x inside this block
    println!("{x}");  // 99
}
println!("{x}");      // 5 — outer x is unaffected
```

### Shadowing vs `mut`

El shadowing permite **cambiar el tipo** de una variable; `mut` no.

```rust
let x = "hello";
let x = x.len();  // ✅ shadowing: &str → usize

let mut y = "hello";
// y = y.len();   // ❌ error: expected &str, found usize (mut does NOT allow type change)
```

> **Punto clave:** Usa shadowing cuando quieras transformar un valor manteniendo el mismo nombre. Usa `mut` cuando necesites reasignar el mismo tipo repetidamente.
