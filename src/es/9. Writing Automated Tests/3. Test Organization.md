# Organización de Pruebas

## Pruebas unitarias

Prueban una función de forma aislada. Se colocan en el mismo archivo, dentro de `mod tests`.

```rust
pub fn add_two(a: i32) -> i32 { a + 2 }

#[cfg(test)]
mod tests {
    use super::*;

    #[test]
    fn it_adds_two() {
        assert_eq!(4, add_two(2));
    }
}
```

## Pruebas de integración

Prueban la API pública. Se colocan en el directorio `tests/` (junto a `src/`).

```rust
// tests/integration_test.rs
use my_crate;

#[test]
fn it_adds_two() {
    assert_eq!(4, my_crate::add_two(2));
}
```

> **Nota:** Las pruebas unitarias pueden probar funciones `privadas`. Las pruebas de integración no.
