# Validating References with Lifetimes

Lifetimes ensure references stay valid as long as needed — prevents **dangling pointers** at compile time.

## The Problem: Dangling Reference

```rust
fn main() {
    let r;

    {
        let x = 5;
        r = &x; // error: `x` does not live long enough
    }

    println!("r: {r}"); // r would point to freed memory
}
```

The compiler catches this at compile time — no runtime panic needed.

## Lifetime Annotation Syntax

Lifetime annotations **describe relationships** between references. They don't change lifetimes.

```rust
// `'a` means: returned ref lives as long as the shortest of the two inputs
fn longest<'a>(x: &'a str, y: &'a str) -> &'a str {
    if x.len() > y.len() { x } else { y }
}
```

- `&'a str` = "a string slice that lives for lifetime `'a`"
- The return type must also be `&'a str` because it could be either `x` or `y`.

## Lifetime in Structs

```rust
struct ImportantExcerpt<'a> {
    part: &'a str,
}

let novel = String::from("Call me Ishmael. Some years ago...");
let first_sentence = novel.split('.').next().expect("Could not find a '.'");
let i = ImportantExcerpt { part: first_sentence };
// `i` cannot outlive `novel` because `part` borrows from it
```

## Lifetime Elision Rules

Rust **implicitly** adds lifetimes in common patterns — no need to write them manually.

| Rule | What it means |
|------|---------------|
| **Each input ref** gets its own lifetime | `fn foo(x: &str)` → `fn foo<'a>(x: &'a str)` |
| **One input ref** → output gets that lifetime | `fn foo(x: &str) -> &str` → `fn foo<'a>(x: &'a str) -> &'a str` |
| **`&self` input** → output gets `&self`'s lifetime | `fn get(&self) -> &str` → `fn get<'a>(&'a self) -> &'a str` |

## Static Lifetime

```rust
let s: &'static str = "I live forever in the binary";
```

- `'static` = the reference lives for the entire program.
- String literals are `&'static str` by default.
- Use sparingly — most of the time you want regular lifetimes.

---

## Key Points

| Concept | Meaning |
|---------|---------|
| **Lifetime `'a`** | Labels a reference's scope; compiler verifies validity. |
| **Dangling pointer** | Reference to deallocated memory — prevented by lifetimes. |
| **Elision** | Compiler fills in lifetimes automatically for common patterns. |
| **`'static`** | Reference lives for the entire program duration. |
