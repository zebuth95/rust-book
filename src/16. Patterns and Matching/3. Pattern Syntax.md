# Pattern Syntax
Patterns are composed of a wide variety of syntax elements, allowing for precise structural matching and conditional binding.

# Pattern Syntax Elements

| Syntax Element    | Description                                    | Example                              |
|-------------------|------------------------------------------------|--------------------------------------|
| Literals          | Match against a specific hardcoded value       | `5`, `'A'`, `true`                   |
| Named Variables   | Bind the matched value to a new variable       | `let x = 5;`, `Some(val)`            |
| `_` (Wildcard)    | Match any value without binding it             | `(_, 5)`                             |
| `..`              | Match an inclusive range of values             | `0..=9`, `'a'..='z'`                 |
| Destructuring     | Break apart structs, enums, tuples, and arrays | `(x, y)`, `Point { x, y: 0 }`, `Color::Red` |
| **`               | `(Or)**                                        | Match against multiple patterns (alternatives)   | `Some(1) | Some(2)`                 |
| `@` (At Operator) | Bind while testing against a pattern           | `Some(id @ 40..=49)`                 |
| Guard Clauses     | Add extra conditional expression to match arm  | `Some(x) if x % 2 == 0 => ...`      |


    #[derive(Debug)]
    enum Message {
        Quit,
        Move { x: i32, y: i32 },
        ChangeColor(i32, i32, i32),
    }
    
    let msg = Message::Move { x: 5, y: 10 };
    
    match msg {
        // Destructuring a struct variant and using a guard clause
        Message::Move { x, y } if x > 0 => {
            println!("Moved right by {} units.", x);
        }
        // Matching a specific tuple (array) variant
        Message::ChangeColor(0, 0, 0) => println!("Black"),
        // Using '@' to bind a value while matching a range
        Message::ChangeColor(r @ 1..=255, _, _) => {
            println!("Color component 'r' is in range: {}", r);
        }
        // Match anything else and bind it to 'other'
        other => println!("Received other message: {:?}", other),
    }

### Literals
Match against specific constant values.

    let x = 5;
    
    match x {
        1 => println!("one"),           // ✅ Match exact number
        2 | 3 => println!("two or three"), // Multiple literals with |
        5 => println!("five"),          // This will match
        _ => println!("something else"),
    }
    
    let grade = 'B';
    match grade {
        'A' => println!("Excellent!"),
        'B' | 'C' => println!("Good"),  // This will match
        'D' => println!("Pass"),
        'F' => println!("Fail"),
        _ => println!("Invalid grade"),
    }
    
    let is_active = true;
        match is_active {
        true => println!("System is active"),
        false => println!("System is inactive"),
    }

### Named Variables
Bind matched values to variables for use.

    let some_option = Some(10);
    
    match some_option {
        Some(value) => println!("Got value: {}", value),  // value = 10
        None => println!("Got nothing"),
    }
    
    let point = (3, 4);
    match point {
        (x, y) => println!("Coordinates: ({}, {})", x, y),  // x=3, y=4
    }
    
    // Variables can shadow outer variables
    let x = 5;
    match Some(10) {
        Some(x) => println!("Inner x: {}", x),  // x = 10 (shadows outer x)
        None => (),
    }
    println!("Outer x: {}", x);  // x = 5

### _ (Wildcard) Pattern
Match any value without binding it.

    let triple = (1, 2, 3);
    
    match triple {
        (1, _, third) => println!("First is 1, third is {}", third),  // third = 3
        _ => println!("Something else"),
    }
    
    let number = 42;
    match number {
        1 => println!("one"),
        _ => (),  // Ignore all other cases without binding
    }
    
    // Useful when you only care about some parts
    let coordinates = (10, 20, 30, 40);
    match coordinates {
        (x, y, _, _) => println!("Position: ({}, {})", x, y),  // Only use x and y
    }

### .. (Range) Patterns
Match inclusive ranges of values.

    let age = 25;
    
    match age {
        0..=12 => println!("Child"),
        13..=19 => println!("Teenager"),
        20..=64 => println!("Adult"),
        65..=120 => println!("Senior"),
        _ => println!("Invalid age"),
    }
    
    let letter = 'm';
    match letter {
        'a'..='z' => println!("Lowercase letter"),
        'A'..='Z' => println!("Uppercase letter"),
        '0'..='9' => println!("Digit"),
        _ => println!("Other character"),
    }
    
    // Numeric ranges
    let score = 85;
    match score {
        90..=100 => println!("A"),
        80..=89 => println!("B"),  // This will match
        70..=79 => println!("C"),
        60..=69 => println!("D"),
        0..=59 => println!("F"),
        _ => println!("Invalid score"),
    }

### Destructuring Patterns
Break apart complex data structures.

#### Destructuring Structs

    struct Point {
        x: i32,
        y: i32,
    }
    
    struct Circle {
        center: Point,
        radius: f64,
    }
    
    let circle = Circle {
        center: Point { x: 10, y: 20 },
        radius: 5.0,
    };
    
    match circle {
        Circle { center: Point { x, y }, radius } => {
            println!("Circle at ({}, {}) with radius {}", x, y, radius);
        }
    }
    
    // Nested destructuring with specific values
    match circle {
        Circle { center: Point { x: 0, y }, radius } => {
            println!("Circle on y-axis at y={}", y);
        }
        Circle { center: Point { x, y: 0 }, radius } => {
            println!("Circle on x-axis at x={}", x);
        }
        _ => println!("Circle not on axis"),
    }

#### Destructuring Enums

    enum Message {
        Quit,
        Move { x: i32, y: i32 },
        Write(String),
        ChangeColor(i32, i32, i32),
    }
    
    let messages = [
        Message::Quit,
        Message::Move { x: 5, y: 10 },
        Message::Write(String::from("hello")),
        Message::ChangeColor(255, 0, 0),
    ];
    
    for msg in messages {
        match msg {
            Message::Quit => println!("Quit message"),
            Message::Move { x, y } => println!("Move to ({}, {})", x, y),
            Message::Write(text) => println!("Text message: {}", text),
            Message::ChangeColor(r, g, b) => println!("Change color to RGB({}, {}, {})", r, g, b),
        }
    }

#### Destructuring Tuples and Arrays

    // Tuple destructuring
    let tuple = (1, "hello", 3.14);
    match tuple {
        (1, s, f) => println!("First is 1, string: {}, float: {}", s, f),
        (x, _, _) => println!("First element is: {}", x),
    }
    
    // Array destructuring (fixed size)
    let array = [1, 2, 3];
    match array {
        [first, second, third] => println!("Array: {}, {}, {}", first, second, third),
        [] => println!("Empty array"),
    }
    
    // Slice patterns (variable length)
    let slice = &[1, 2, 3][..];
    match slice {
        [1, rest @ ..] => println!("Starts with 1, rest: {:?}", rest),
        [first, middle @ .., last] => println!("First: {}, middle: {:?}, last: {}", first, middle, last),
        [] => println!("Empty slice"),
    }

### | (Or) Pattern
Match against multiple alternatives.

    let dice_roll = 4;
    
    match dice_roll {
        1 | 3 | 5 => println!("Odd number"),
        2 | 4 | 6 => println!("Even number"),  // This will match
        _ => println!("Invalid dice roll"),
    }
    
    enum TrafficLight {
        Red,
        Yellow,
        Green,
    }
    
    let light = TrafficLight::Yellow;
    match light {
        TrafficLight::Red | TrafficLight::Yellow => println!("Stop or caution"),  // This will match
        TrafficLight::Green => println!("Go"),
    }
    
    // Complex OR patterns
    let value = (1, 2);
    match value {
        (1, _) | (_, 1) => println!("Contains 1"),  // This will match
        (2, 2) => println!("Both are 2"),
        _ => println!("Something else"),
    }

### @ (At Operator)
Bind the value to a variable while simultaneously testing it.

    let age = 25;
    
    match age {
        age @ 0..=17 => println!("Minor (age: {})", age),
        age @ 18..=64 => println!("Adult (age: {})", age),  // This will match: age = 25
        age @ 65..=120 => println!("Senior (age: {})", age),
        _ => println!("Invalid age"),
    }
    
    enum Message {
        Login { id: i32, name: String },
        Logout { id: i32 },
    }
    
    let msg = Message::Login { id: 42, name: "Alice".to_string() };
    
    match msg {
        Message::Login { id: user_id @ 40..=49, name } => {
            println!("User {} (ID: {}) logged in", name, user_id);  // This will match
        }
        Message::Login { id, name } => {
            println!("User {} (ID: {}) logged in", name, id);
        }
        Message::Logout { id } => {
            println!("User ID {} logged out", id);
        }
    }
    
    // Complex @ patterns
    let point = (5, 10);
    match point {
        (x @ 1..=5, y @ 6..=10) => {
            println!("Point in quadrant: ({}, {})", x, y);  // This will match
        }
        (x, y) => println!("Point: ({}, {})", x, y),
    }

### Guard Clauses
Add extra conditions to match arms.

    let number = Some(15);
    
    match number {
        Some(x) if x < 10 => println!("Less than 10: {}", x),
        Some(x) if x % 2 == 0 => println!("Even number: {}", x),
        Some(x) if x > 10 && x < 20 => println!("Between 10 and 20: {}", x),  // This will match
        Some(x) => println!("Other number: {}", x),
        None => println!("No number"),
    }
    
    // Multiple conditions in guards
    let user = ("admin", "password123");
    
    match user {
        (username, password) if username == "admin" && password.len() >= 8 => {
            println!("Admin login successful");
        }
        (username, _) if username == "guest" => {
            println!("Guest login");
        }
        _ => println!("Login failed"),
    }
    
    // Guards with destructuring
    enum Shape {
        Circle { radius: f64 },
        Rectangle { width: f64, height: f64 },
    }
    
    let shape = Shape::Rectangle { width: 10.0, height: 5.0 };
    
    match shape {
        Shape::Circle { radius } if radius > 0.0 => {
            println!("Circle with positive radius: {}", radius);
        }
        Shape::Rectangle { width, height } if width == height => {
            println!("Square with side: {}", width);
        }
        Shape::Rectangle { width, height } if width > height => {
            println!("Landscape rectangle: {}x{}", width, height);  // This will match
        }
        Shape::Rectangle { width, height } => {
            println!("Portrait rectangle: {}x{}", width, height);
        }
        _ => println!("Invalid shape"),
    }

## Key Pattern Matching Rules
- Patterns are exhaustive - match must cover all possibilities
- Order matters - Patterns are checked from top to bottom
- Variables shadow outer ones - Be careful with naming
- Use _ to ignore values you don't need
- Guards add extra conditions beyond structural matching
- @ binds and tests simultaneously - powerful for complex conditions

Patterns make Rust's matching incredibly expressive and safe, ensuring you handle all possible cases while providing flexible ways to extract and test data!
