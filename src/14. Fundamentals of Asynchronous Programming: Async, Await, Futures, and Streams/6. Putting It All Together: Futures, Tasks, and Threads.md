# Putting It All Together: Futures, Tasks, and Threads
Task A Task is a lightweight, conceptual unit of work that contains a single Future. It is the basic unit scheduled by the async runtime.

- Thread (OS): An operating system resource; typically a high overhead.
- Task (Runtime): A lightweight future managed by the executor; many tasks run on one thread.
- Future (Code): The state machine generated by async that defines the task's work.

## Execution:
A task is executed by an Executor (the runtime) which assigns it to an OS thread. When a task hits an await and is not ready, the task is suspended (put to sleep), and the thread immediately switches to poll another waiting task. This efficient switching is what gives async its high throughput.
