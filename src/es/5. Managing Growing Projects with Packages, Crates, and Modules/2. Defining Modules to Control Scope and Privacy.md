# Definiendo Módulos para Controlar el Ámbito y la Privacidad
- Los **Módulos** ayudan a organizar el código dentro de un crate agrupando funcionalidades relacionadas.
También controlan la **privacidad** de los elementos, es decir, qué partes de tu código son accesibles desde el código externo.
- La palabra clave **mod** se utiliza para declarar un módulo. Los módulos pueden estar anidados unos dentro de otros.
- Por defecto, todos los elementos (funciones, estructuras, enumeraciones, etc.) son privados dentro del módulo.
Para hacer que un elemento sea público y accesible desde un módulo padre, debes usar la palabra clave **pub**.


### Ejemplo:
Este código muestra un módulo `garden` que contiene un módulo `vegetables`, el cual contiene una estructura `Asparagus` pública.

    // En src/lib.rs
    pub mod garden {
        pub mod vegetables {
            pub struct Asparagus {
                pub name: String,
            }
        }
    }

En el ejemplo anterior, la estructura `Asparagus` y su campo `name` se hacen públicos con la palabra clave `pub` para que puedan usarse fuera de sus módulos.
