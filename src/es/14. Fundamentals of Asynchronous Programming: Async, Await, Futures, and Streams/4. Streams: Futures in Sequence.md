# Streams: Futures en Secuencia

Un **Stream** es un **iterador asíncrono** — los valores llegan uno a la vez a lo largo del tiempo (paquetes de red, filas de BD, lecturas de sensores).

| Trait | Método | Salida |
|-------|--------|--------|
| `Iterator` | `.next()` | `Option<Item>` |
| `Stream` | `.next().await` | `Poll<Option<Item>>` |

## Uso Básico de Stream

```rust
use tokio_stream::StreamExt;

async fn process(stream: impl Stream<Item = i32>) {
    tokio::pin!(stream);
    while let Some(item) = stream.next().await {
        println!("Got: {item}");
    }
}
```

## Creando un Stream

```rust
use tokio::time::{sleep, Duration};
use tokio_stream::StreamExt;
use futures::stream;

#[tokio::main]
async fn main() {
    // Create a stream that yields numbers 1..=5 with 100ms delay each
    let s = stream::iter(1..=5).then(|n| async move {
        sleep(Duration::from_millis(100)).await;
        n
    });

    tokio::pin!(s);
    while let Some(n) = s.next().await {
        println!("Received: {n}"); // Each item arrives ~100ms apart
    }
}
```

## Combinadores de Stream

```rust
use tokio_stream::StreamExt;

let s = stream::iter(1..=10);

s.map(|n| n * 2)           // transform each item
 .filter(|n| *n > 5)       // keep only items matching a predicate
 .take(3)                   // take only first 3 items
 .collect::<Vec<_>>().await // collect all into a collection
```

## Combinadores Clave

| Método | Qué hace |
|--------|-------------|
| `.next().await` | Obtiene el siguiente elemento (espera a que esté listo) |
| `.map(f)` | Transforma cada elemento |
| `.filter(p)` | Mantiene elementos que cumplen un predicado |
| `.take(n)` | Toma los primeros `n` elementos |
| `.collect()` | Recolecta en una colección |
| `.then(f)` | Aplica una función asíncrona a cada elemento |

---

## Puntos Clave

| Concepto | Significado |
|---------|---------|
| **Stream** | Como `Iterator` pero cada `.next()` puede necesitar hacer await. |
| **`tokio_stream::StreamExt`** | Proporciona `.next()` y otros combinadores en streams. |
| **`tokio::pin!`** | Fija el stream en memoria (necesario porque `Stream` incluye autoreferencias). |
| **Perezoso** | Los streams no hacen trabajo hasta que se les hace poll. |
