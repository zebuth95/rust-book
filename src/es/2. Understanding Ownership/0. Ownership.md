# Propiedad (Ownership)

**La Propiedad (Ownership)** es la forma en que Rust gestiona la memoria sin un recolector de basura. Tres reglas:

1. Cada valor tiene **un único dueño**.
2. Solo **un dueño a la vez**.
3. Cuando el dueño sale del alcance, el valor se **elimina**.

## Stack vs Heap

| Stack | Heap |
|---|---|
| Tamaño fijo, LIFO | Tamaño dinámico |
| Rápido | Más lento (asignación + seguimiento de puntero) |
| Variables locales | Box, Vec, String |

## Propiedad y Movimientos (Ownership & Moves)

```rust
let s1 = String::from("hello");
let s2 = s1;          // s1 MOVED to s2
// println!("{s1}");  // ❌ error: s1 no longer valid
```

## Clone (copia profunda)

```rust
let s1 = String::from("hello");
let s2 = s1.clone();   // ✅ deep copy
println!("{s1}, {s2}");
```

## Tipos Copy (solo stack)

Los escalares y las tuplas de escalares implementan `Copy` — se copian automáticamente.

```rust
let x = 5;
let y = x;            // x is Copy, still valid
println!("{x}, {y}");  // ✅
```

## Referencias y Préstamo (References & Borrowing)

Una referencia te permite usar un valor sin tomar posesión.

```rust
fn main() {
    let s1 = String::from("hello");
    let len = calculate_length(&s1);  // borrow, not move
    println!("'{s1}' length = {len}");  // s1 still usable
}

fn calculate_length(s: &String) -> usize {
    s.len()
}
```

### Reglas del Préstamo (Borrowing)

- Puedes tener **cualquier número de referencias inmutables** (`&T`).
- O **exactamente una referencia mutable** (`&mut T`).
- Las referencias deben **ser siempre válidas** (sin punteros colgantes).

```rust
let mut s = String::from("hello");
let r1 = &s;          // ✅ multiple immutable OK
let r2 = &s;          // ✅
// let r3 = &mut s;   // ❌ can't borrow as mutable while immutable exists
```

## Slice (Rebanadas)

Un slice es una **referencia** a una secuencia contigua (sin propiedad).

```rust
let arr = [1, 2, 3, 4, 5];
let slice = &arr[1..3];         // [2, 3]

let s = String::from("hello world");
let hello = &s[0..5];           // "hello"
let world = &s[6..];            // "world"
```

> **Nota:** `&str` es un slice de cadena — el tipo de cadena más común en parámetros de función.

## Vida Útil No Léxica (NLL)

El verificador de préstamos de Rust es inteligente — los préstamos terminan cuando la referencia se **usa por última vez**, no cuando termina el ámbito.

```rust
let mut x = 5;
let y = &mut x;
*y += 1;
// y is no longer used here — borrow ends (NLL)
let z = &x;   // ✅ allowed because y's borrow ended
println!("{z}");
```

Esto se llama **Vida Útil No Léxica** (NLL), estabilizada en la edición Rust 2018.

## Reglas de Elisión de Vida Útil (Detalladas)

Tres reglas que el compilador aplica automáticamente para reducir la carga de anotaciones:

1. **Cada** referencia de entrada obtiene su propio parámetro de vida útil.
2. Si hay **exactamente una** vida útil de entrada, se asigna a todas las referencias de salida.
3. Si `&self` o `&mut self` es una entrada, su vida útil se asigna a todas las referencias de salida.

```rust
// Rule 1: fn foo(x: &str, y: &str) → fn foo<'a, 'b>(x: &'a str, y: &'b str)
// Rule 2: fn foo(x: &str) -> &str → fn foo<'a>(x: &'a str) -> &'a str
// Rule 3: fn foo(&self, x: &str) -> &str → fn foo<'a, 'b>(&'a self, x: &'b str) -> &'a str
```

## Varianza de Vida Útil

La varianza describe cómo las relaciones de subtipificación se propagan a través de los constructores de tipos.

- **Covariante** — `&'a T` es un subtipo de `&'b T` si `'a` sobrevive a `'b` (es decir, `'a: 'b`).
- **Invariante** — `&'a mut T` es invariante: no se puede sustituir por una vida útil más larga.
- La mayoría de los tipos son covariantes en sus parámetros de vida útil. `Cell<T>` y `&mut T` son invariantes.
- La varianza permite la mutación segura a través de referencias.

> **Resumen:** Ownership, borrowing, lifetimes y variance forman la base de la seguridad de memoria de Rust. Ownership determina cuándo se eliminan los valores. Borrowing (referencias) permite acceso temporal sin tomar posesión. Los parámetros de vida útil (`'a`) conectan los valores prestados con sus orígenes, y las reglas de elisión minimizan las anotaciones que necesitas escribir. NLL hace que el verificador de préstamos sea más permisivo al liberar los préstamos en su último uso. La varianza asegura que las relaciones de subtipificación (especialmente en torno a las vidas útiles) se mantengan sólidas, siendo `&mut T` y `Cell<T>` invariantes para prevenir violaciones de aliasing. Juntos, estos conceptos permiten a Rust garantizar la seguridad de memoria en tiempo de compilación — sin recolector de basura, sin punteros colgantes, sin condiciones de carrera.
