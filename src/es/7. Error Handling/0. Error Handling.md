# Manejo de Errores

Dos categorías de errores:

| Tipo | Qué es | Cómo manejarlo |
|---|---|---|
| **Recuperable** | Archivo no encontrado, tiempo de espera de red | `Result<T, E>` |
| **Irrecuperable** | Índice fuera de límites en un arreglo, bug | `panic!` |

Rust **no tiene excepciones**. Usa `Result` para fallos esperados, `panic!` para bugs.

## Ecosistema de Crates para Errores

| Crate | Caso de Uso |
|-------|-------------|
| **`thiserror`** | Derivar `Error` para tipos de error en librerías (enums personalizados) |
| **`anyhow`** | Manejo de errores ad-hoc en aplicaciones (azúcar sintáctico para `Box<dyn Error>`) |
| **`eyre`** | Reporte de errores flexible con informes personalizables (como anyhow pero más configurable) |

### Ejemplo con `thiserror` (Librerías)

```rust
use thiserror::Error;

#[derive(Error, Debug)]
pub enum DataError {
    #[error("data store disconnected")]
    Disconnect(#[from] io::Error),
    #[error("invalid header: {0:?}")]
    InvalidHeader(Vec<u8>),
    #[error("unknown error")]
    Unknown,
}
```

### Ejemplo con `anyhow` (Aplicaciones)

```rust
use anyhow::{Result, Context};

fn read_config() -> Result<String> {
    let content = std::fs::read_to_string("config.toml")
        .with_context(|| "failed to read config file")?;
    Ok(content)
}
```

> **Nota:** Usa `thiserror` en librerías (quien llama decide cómo manejar). Usa `anyhow` en aplicaciones (solo reporta y continúa).
