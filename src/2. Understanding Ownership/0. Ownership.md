# Ownership

**Ownership** is Rust's way of managing memory without a garbage collector. Three rules:

1. Each value has **one owner**.
2. Only **one owner at a time**.
3. When the owner goes out of scope, the value is **dropped**.

## Stack vs Heap

| Stack | Heap |
|---|---|
| Fixed size, LIFO | Dynamic size |
| Fast | Slower (allocation + pointer chase) |
| Local variables | Box, Vec, String |

## Ownership & Moves

```rust
let s1 = String::from("hello");
let s2 = s1;          // s1 MOVED to s2
// println!("{s1}");  // ❌ error: s1 no longer valid
```

## Clone (deep copy)

```rust
let s1 = String::from("hello");
let s2 = s1.clone();   // ✅ deep copy
println!("{s1}, {s2}");
```

## Copy types (stack-only)

Scalars and tuples of scalars implement `Copy` — automatically copied.

```rust
let x = 5;
let y = x;            // x is Copy, still valid
println!("{x}, {y}");  // ✅
```

## References & Borrowing

A reference lets you use a value without taking ownership.

```rust
fn main() {
    let s1 = String::from("hello");
    let len = calculate_length(&s1);  // borrow, not move
    println!("'{s1}' length = {len}");  // s1 still usable
}

fn calculate_length(s: &String) -> usize {
    s.len()
}
```

### Rules of Borrowing

- You can have **any number of immutable references** (`&T`).
- OR **exactly one mutable reference** (`&mut T`).
- References must **always be valid** (no dangling pointers).

```rust
let mut s = String::from("hello");
let r1 = &s;          // ✅ multiple immutable OK
let r2 = &s;          // ✅
// let r3 = &mut s;   // ❌ can't borrow as mutable while immutable exists
```

## Slices

A slice is a **reference** to a contiguous sequence (no ownership).

```rust
let arr = [1, 2, 3, 4, 5];
let slice = &arr[1..3];         // [2, 3]

let s = String::from("hello world");
let hello = &s[0..5];           // "hello"
let world = &s[6..];            // "world"
```

> **Note:** `&str` is a string slice — the most common string type in function params.
