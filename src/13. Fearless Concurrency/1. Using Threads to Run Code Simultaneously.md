# Using Threads to Run Code Simultaneously
Threads are the mechanism for running multiple sequences of instructions concurrently.
Rust use the **on-to-one threading model (one OS thread per language thread).

## Implementation
The standard library's **std::thread::spawn** function creates a new thread, The function takes a **closure** (Anonymous Functions)
as an argument, which contains the code to be run in the new thread.

## Ownership and Moves
To ensure the thread's code outlives any data it uses, the closure often requires the **move** keyword.
This  forces the closure to take ownership of the data it captures from the environment, guaranteeing the data is valid for the entire
life of the thread.

## Coordination
The **join()** method on the **JoinHandle** (returned by thread::spawn) blocks the execution of the spawning until the 
thread has finished.

### (returned by thread::spawn) 

    use std::thread;
    use std::time::Duration;
    
    let handle = thread::spawn(|| {
    for i in 1..5 {
        println!("hi number {} from the spawned thread!", i);
        thread::sleep(Duration::from_millis(1));
    }
    });
    
    // The main thread waits for the spawned thread to complete.
    handle.join().unwrap();
