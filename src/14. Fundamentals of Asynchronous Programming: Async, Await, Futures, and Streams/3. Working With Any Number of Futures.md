# Working With Any Number of Futures
When you have independent futures, you use combinator functions to manage their execution.

- **join!** (or similar combinators): Runs multiples futures concurrently  and waits for all of them to complete.
  this is ideal when all results are required.
- **select!** (or similar combinators): Runs multiple futures concurrently but return as soon as the first one completes.
  This is useful for time-outs or races.

## Implementation:
These macros/functions are provided by runtime crates (e.g., tokio::join!, tokio::select!).

## join!

    use tokio::time::sleep;
    use std::time::Duration;

    #[tokio::main]
    async fn main() {
        start_work().await;
    }
    
    // Imagine this is running on an executor like Tokio
    async fn start_work() {
        let f1 = async {
        // This task starts
        println!("Task 1: Started");
        sleep(Duration::from_millis(100)).await;
        println!("Task 1: Finished");
        };
    
        let f2 = async {
            // This task starts almost simultaneously, before f1 finishes
            println!("Task 2: Started");
            sleep(Duration::from_millis(50)).await;
            println!("Task 2: Finished");
        };
    
        // Tasks run concurrently, interleaved by the executor.
        // tokio::join! executes the futures concurrently and waits for all of them.
        tokio::join!(f1, f2);
    }

## select!

    use tokio::time::{sleep, Duration};
    
    async fn fast_task() -> &'static str {
        // Simulate a fast operation
        sleep(Duration::from_millis(100)).await;
        "Fast Task Completed"
    }
    
    async fn slow_task() -> &'static str {
        // Simulate a slow operation
        sleep(Duration::from_millis(500)).await;
        "Slow Task Completed"
    }
    
    async fn race_futures() {
        println!("Starting race between fast_task and slow_task...");

        // The select! macro runs both futures concurrently and returns 
        // as soon as one of them completes.
        let result = tokio::select! {
            // The pattern is: pattern = future => expression
            // Here, 'result_fast' will be bound to the value returned by fast_task()
            // if it completes first. The block { ... } is then executed.
            result_fast = fast_task() => {
                // This arm runs if fast_task() completes first.
                format!("The winner is: {}", result_fast)
            },
            // 'result_slow' will be bound to the value returned by slow_task() 
            // if it completes first (which shouldn't happen here).
            result_slow = slow_task() => {
                // This arm runs if slow_task() completes first.
                format!("The winner is: {}", result_slow)
            },
        };

        // The output will show the result from the 'fast_task'
        println!("{}", result); 
    }
    
    // How to run this in a main function:
    #[tokio::main]
    async fn main() {
        race_futures().await;
        // Expected output will mention "Fast Task Completed"
    }
