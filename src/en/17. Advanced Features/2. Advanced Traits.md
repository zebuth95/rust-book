# Advanced Traits

## Associated Types

```rust
pub trait Iterator {
    type Item;
    fn next(&mut self) -> Option<Self::Item>;
}
```

## Operator Overloading

```rust
use std::ops::Add;

#[derive(Debug, PartialEq)]
struct Point { x: i32, y: i32 }

impl Add for Point {
    type Output = Point;
    fn add(self, other: Point) -> Point {
        Point { x: self.x + other.x, y: self.y + other.y }
    }
}
```

## Supertraits

```rust
trait Animal { fn name(&self) -> &str; }
trait Canine: Animal {  // Canine requires Animal
    fn bark(&self) { println!("{} says woof!", self.name()); }
}
```

## Fully Qualified Syntax

```rust
trait Pilot { fn fly(&self); }
trait Wizard { fn fly(&self); }
struct Human;
impl Human { fn fly(&self) { println!("waving arms"); } }

let person = Human;
Pilot::fly(&person);   // call trait method explicitly
Wizard::fly(&person);
```

> **Note:** Use `Trait::method(&self)` when there's a naming conflict.

#### Generic Associated Types (GATs)

GATs allow associated types to have their own generic parameters. Stabilized in Rust 1.65.

```rust
trait Iterable {
    type Item<'a>;
    type Iter<'a>: Iterator<Item = Self::Item<'a>> where Self: 'a;

    fn iter<'a>(&'a self) -> Self::Iter<'a>;
}

impl<T: 'static> Iterable for Vec<T> {
    type Item<'a> = &'a T;
    type Iter<'a> = std::slice::Iter<'a, T>;

    fn iter<'a>(&'a self) -> Self::Iter<'a> {
        self.as_slice().iter()
    }
}
```

Use cases:
- Lending iterators (iterator yields references tied to the borrow).
- Async traits with associated futures.

> **Note:** GATs enable expressive patterns like lending iterators and async traits that were previously impossible without boxing.
