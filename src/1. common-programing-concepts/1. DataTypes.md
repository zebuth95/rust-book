# Data Types

Every value has a type. Two categories: **scalar** and **compound**.

## Scalar Types

| Type | Examples |
|---|---|
| Integers | `i8`, `u8`, `i32` (default), `u64`, `isize`, `usize` |
| Floats | `f32`, `f64` (default) |
| Bool | `true`, `false` |
| Char | `'a'`, `'😻'` (4 bytes, Unicode) |

```rust
let a: i32 = -10;      // signed
let b: u8 = 255;       // unsigned
let c = 2.0;           // f64
let d: f32 = 3.0;      // f32
let t = true;
let heart = '😻';
```

## Signed vs Unsigned Integers

| | Signed (`i`) | Unsigned (`u`) |
|---|---|---|
| **Range** | Includes negatives | Zero and positive only |
| **Example** | `i8`: **-128** to **127** | `u8`: **0** to **255** |
| **Naming** | `i` + bits (`i8`, `i16`, `i32`, `i64`, `i128`, `isize`) | `u` + bits (`u8`, `u16`, `u32`, `u64`, `u128`, `usize`) |
| **When to use** | Values can be negative (temperature, balance) | Values are always non-negative (counts, IDs, sizes) |

- `isize` / `usize` depend on your CPU architecture (32-bit or 64-bit).
- **Default integer type** is `i32` — good balance of speed and range.
- Overflow behavior differs: debug mode **panics**, release mode **wraps** (two's complement).

```rust
let signed: i8 = -128;        // min i8
let unsigned: u8 = 255;       // max u8
// let oops: u8 = -5;         // compile error: can't assign negative to unsigned

let arch_size: usize = 8;     // 8 on 64-bit machines
```

## Compound Types

### Tuple — fixed length, mixed types

```rust
let tup: (i32, f64, u8) = (500, 6.4, 1);
let (x, y, z) = tup;          // destructuring
let first = tup.0;             // index access
```

### Array — fixed length, same type

```rust
let arr: [i32; 3] = [1, 2, 3];
let first = arr[0];   // index
```

> **Note:** Use `Vec` (heap-allocated) for growable arrays.
