# Closures: Anonymous Functions That Capture Their Environment
Rust's closures are anonymous functions you can save in a variable or pass as argument to other functions.
Unlike regular functions, they can capture and access variable from the scope in which they are defined.

### They are defined by what data they capture:

- Capturing by Borrowing (&T or &mut T): The most common way, similar to borrowing a reference.
- Capturing by Moving (T): Moves the captured value into the closure's definition, often required when passing a closure to a new thread. The move keyword forces this behavior.


    |val| val + x


| Trait     | Capture Mechanism     | Description | Call Count Limit |
|-----------|-----------------------|---|------------------|
| **Fn**    | Immutable Borrow (&T) | Only reads the captured variable | Multiple times   |
| **FnMut** | Mutable Borrow (&mut T) | Modifies the captured variable | Multiple times |
| **FnOnce** | Consume/Move (T) | Takes ownership of (consumes) the captured variable.	 | Only once |


### Example 1: Fn (Capturing by Immutable Reference)
A closure implements Fn if it only requires an immutable reference to the captured variable.

    fn main() {
    let greeting = String::from("Hello");
    
        // This closure only reads 'greeting', so it implements Fn.
        let print_hello = |name| {
            println!("{}, {}!", greeting, name);
        };
    
        // 'greeting' is still usable here because it was only borrowed.
        print_hello("Alice");
        print_hello("Bob");
        println!("Original greeting is still: {}", greeting);
    }

### Example 2: FnMut (Capturing by Mutable Reference)
A closure implements FnMut if it mutates the captured variable. The closure variable itself must be mut.

    fn main() {
    let mut counter = 0;
    
        // This closure mutates 'counter', so it implements FnMut.
        let mut add_one = || {
            counter += 1; // Mutating the captured variable
            println!("Counter is now: {}", counter);
        };
    
        // We can call it multiple times.
        add_one(); // Output: Counter is now: 1
        add_one(); // Output: Counter is now: 2
    }

### Example 3: FnOnce (Capturing by Value/Consuming)
A closure implements FnOnce if it takes ownership of a captured variable, often because it moves the variable into its body (e.g., using the move keyword or consuming it with a method like std::mem::drop). Since it consumes the data, it can only be called once.

    fn main() {
    let data_vec = vec![1, 2, 3];
    
        // The 'move' keyword forces capture by value (FnOnce).
        // The closure takes ownership of 'data_vec'.
        let consume_vec = move || {
            // We drop (consume) the data inside the closure.
            println!("Processing vector of length: {}", data_vec.len());
            std::mem::drop(data_vec);
        };
    
        // The first call consumes the data.
        consume_vec();
    
        // ERROR: This second call would fail to compile because
        // 'data_vec' has been consumed by the first call.
        // consume_vec();
    }
