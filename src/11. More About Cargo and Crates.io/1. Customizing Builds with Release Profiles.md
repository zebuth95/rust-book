# Customizing Builds with Release Profiles
**Release Profiles** are pre-defined or custom configurations that control various compilation options, primarily
optimization and debugging symbols. they allow you to tune your code for specific environments.

## Default Profiles: Cargo ships with two main profiles:

- dev (Default for cargo build): Fast to compile, with minimal optimizations and inclusion of debugging symbols. Ideal for quick iteration during development.
- release (Default for cargo build --release): Slow to compile, with maximum optimizations and no debugging symbols. Ideal for production code.

### Customization:
You can define or override profiles in Cargo.toml

    [profile.dev]
    opt-level = 0      # Default: No optimization for fast compilation
    
    [profile.release]
    opt-level = 3      # Default: Heavy optimization for maximum performance
    
    [profile.testing]  # Custom profile for highly optimized test builds
    inherits = "dev"
    opt-level = 2
