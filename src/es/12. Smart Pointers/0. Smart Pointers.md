# Punteros Inteligentes (Smart Pointers)

Los punteros inteligentes poseen datos y actúan como referencias (mediante `Deref`) con lógica de limpieza (mediante `Drop`).

| Puntero | Descripción |
|---|---|
| `Box<T>` | Asignación en el heap |
| `Rc<T>` | Conteo de referencias (monohilo, propiedad compartida) |
| `RefCell<T>` | Verificación de préstamos en tiempo de ejecución (mutabilidad interior) |
| `Weak<T>` | Referencia sin propiedad, previene ciclos |
| `Cow<T>` | Clone-on-write — toma prestado o posee datos de forma diferida |

## `Cow<T>` (Clone-on-Write)

Un enum que toma prestado (`Cow::Borrowed`) o posee (`Cow::Owned`) datos. Clona solo cuando es necesario mutar.

```rust
use std::borrow::Cow;

fn normalize(name: &str) -> Cow<str> {
    if name.contains(' ') {
        // Necesita modificación — asigna un String propio
        name.replace(' ', "_").into()
    } else {
        // No necesita modificación — devuelve prestado
        Cow::Borrowed(name)
    }
}

let simple = normalize("hello");      // Cow::Borrowed, sin asignación
let spaced = normalize("hello world"); // Cow::Owned, asignado una vez
```

- **Úsalo para**: Funciones que a veces modifican la entrada, a veces la devuelven tal cual (costo cero).
- **Métodos clave**: `.to_mut()`, `.into_owned()`.

## `Pin<T>` — Fijación (Pinning)

`Pin<Box<T>>` o `Pin<&mut T>` garantiza que el valor **no se moverá en memoria** después de ser fijado.

```rust
use std::pin::Pin;

let data = String::from("hello");
let pinned: Pin<Box<String>> = Box::pin(data);
// pinned.as_ref().get_ref() — acceso inmutable
```

- **Requerido** para estructuras auto-referenciales (máquinas de estado asíncronas).
- `Pin<&mut T>` es el tipo recibido en `Future::poll()`.
