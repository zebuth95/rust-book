# Macros

**Metaprogramación** — código que escribe código.

## Macros Declarativas (`macro_rules!`)

```rust
macro_rules! say_hello {
    () => { println!("Hello!"); };
    ($name:expr) => { println!("Hello, {}!", $name); };
}

say_hello!();
say_hello!("Alice");
```

## Repetición

```rust
macro_rules! my_vec {
    ($($x:expr),*) => {
        {
            let mut v = Vec::new();
            $(v.push($x);)*
            v
        }
    };
}

let v = my_vec![1, 2, 3];
```

## Macros Procedurales

- `#[derive(Debug)]` — derive personalizado
- `#[attribute]` — tipo atributo
- `custom!()` — tipo función

```rust
use serde::Deserialize;

#[derive(Deserialize)]   // custom derive macro
struct Config { name: String }
```

> **Nota:** Usa `macro_rules!` para generación de código simple. Usa macros procedurales para transformaciones complejas.

#### Higiene de Macros

Las macros son **higiénicas** — los identificadores declarados dentro de una macro no se filtran al ámbito de llamada.

```rust
macro_rules! create_fn {
    ($name:ident) => {
        fn $name() {
            println!("called {}", stringify!($name));
        }
    };
}

create_fn!(foo);
foo(); // prints "called foo"
```

#### `$crate` — Referenciando el Crate Actual

Usa siempre `$crate::` para referirte a elementos del crate donde se define la macro para evitar problemas de resolución de nombres.

```rust
// In the crate where the macro is defined
macro_rules! my_error {
    ($msg:expr) => {
        $crate::MyError::new($msg)
    };
}
```

#### Macros Procedurales — Ejemplo de Derive Macro

```rust
// my_derive/src/lib.rs
use proc_macro::TokenStream;

#[proc_macro_derive(MyTrait)]
pub fn my_trait_derive(input: TokenStream) -> TokenStream {
    // Parse the input and generate impl code
    input  // simplified — real impl uses syn + quote crates
}
```

> **Nota:** Usa `macro_rules!` para generación de código simple. Usa macros procedurales (con los crates `syn` y `quote`) para transformaciones complejas.
