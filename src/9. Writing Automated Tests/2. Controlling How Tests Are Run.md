# Controlling How Tests Are Run
Just as **cargo run**  compiles your code and then the resultant binary, **cargo test** compiles  your code in test
mode and runs the resultant test binary.


## Running Tests in Parallel or Consecutively
When you run multiple tests, by default they run in parallel using threads, meaning they finish running faster and you
get feedback quicker.

If you don’t want to run the tests in parallel or if you want more fine-grained control over the number of threads used, you can send the --test-threads flag and the number of threads you want to use to the test binary. Take a look at the following example:

    $ cargo test -- --test-threads=1

cargo test supports various command-line options for granular control over test execution, which is vital for large codebases and CI/CD pipelines.

## Filtering:
You can pass a string argument to cargo test to run only tests whose names contain that string (e.g., cargo test addition).

## Ignoring Tests:
The #[ignore] attribute allows you to skip slow or expensive tests during a normal run. To run only ignored tests, use cargo test -- --ignored.

## Test Output:
By default, Rust only prints output (like println!) for failed tests. The --nocapture flag forces all test output to be displayed.

## Running Options:
Flags like --test-threads=1 (to run tests sequentially) and --no-fail-fast (to run all tests even after a failure) provide control over concurrency and execution flow.
