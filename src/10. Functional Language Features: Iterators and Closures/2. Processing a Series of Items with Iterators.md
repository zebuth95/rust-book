# Processing a Series of Items with Iterators
Iterator are a standard and essential mechanism for processing sequences of items in Rust.
They implement **Iterator trait**, which requires the **next()** method.
**next()** is a **consuming adapter** because it consumes the item and mutates the iterator's
internal state to point to the next element.

- **lazy Evaluation**: iterators are lazy; they chain of processing (map, filter) does nothing until a
consuming adapter (collect, sum, next()) is called.

### Example (Lazy vs. Consuming):

    let v1 = vec![1, 2, 3];
    let v1_iter = v1.iter(); // Iterator is created
    
    // `map` is an **iterator adapter** (lazy). It returns a new iterator.
    let mapped_iter = v1_iter.map(|x| x + 1);
    println!("The iterator has been created, but nothing has been calculated yet.");
    
    // `collect` is a **consuming adapter**. It forces execution.
    let v2: Vec<_> = mapped_iter.collect();
    assert_eq!(v2, vec![2, 3, 4]);
