# Cargo Workspaces
A workspace is a set of packages (crates) that share the same Cargo.lock and output directory. It's the standard way to manage large projects composed of tightly integrated sub-crates (e.g., a library and a binary that uses it).

- Benefits: Simplified coordination (a single cargo build builds all members), consistent dependencies (shared Cargo.lock), and easier development between local crates.
- Structure: A workspace is defined by a root Cargo.toml file that does not contain a [package] section but lists its members under [workspace].


### Example (Workspace Structure):

    .
    ├── Cargo.toml      # [workspace] file lists members
    ├── target          # Single, shared output directory
    ├── adder           # Member 1: A binary crate
    │   └── src
    └── add_one         # Member 2: A library crate used by 'adder'
    └── src
