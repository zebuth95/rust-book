# A Closer Look at the Traits for Async

The entire async system is built on a few key traits.

## The `Future` Trait

```rust
use std::pin::Pin;
use std::task::{Context, Poll};

trait Future {
    type Output;

    fn poll(self: Pin<&mut Self>, cx: &mut Context<'_>) -> Poll<Self::Output>;
}
```

| Piece | Purpose |
|-------|---------|
| **`Output`** | The type produced when the future completes. |
| **`Pin<&mut Self>`** | Prevents the future from being moved in memory — critical for self-referential state machines. |
| **`Context` / `Waker`** | The future registers a waker so the executor knows to poll again when data is ready. |
| **`Poll::Ready(T)`** | Computation finished with value `T`. |
| **`Poll::Pending`** | Not ready yet — call `poll()` again when woken. |

## How `async` Desugars

The compiler transforms `async fn` into a struct + `Future` impl:

```rust
// This:
async fn example() -> u32 {
    let x = 1;
    let y = 2;
    x + y
}

// Roughly becomes:
enum ExampleStateMachine {
    Start,
    Done(u32),
}

impl Future for ExampleStateMachine {
    type Output = u32;
    fn poll(self: Pin<&mut Self>, _cx: &mut Context<'_>) -> Poll<u32> {
        match *self {
            ExampleStateMachine::Start => {
                let x = 1;
                let y = 2;
                let result = x + y;
                *self = ExampleStateMachine::Done(result);
                Poll::Ready(result)
            }
            ExampleStateMachine::Done(result) => Poll::Ready(result),
        }
    }
}
```

## The `Stream` Trait

```rust
trait Stream {
    type Item;
    fn poll_next(self: Pin<&mut Self>, cx: &mut Context<'_>) -> Poll<Option<Self::Item>>;
}
```

- Same pattern as `Future` but yields **multiple items** over time.
- `Poll::Ready(Some(item))` — next value available.
- `Poll::Ready(None)` — stream exhausted.
- `Poll::Pending` — waiting for next value.

---

## Key Points

| Concept | Meaning |
|---------|---------|
| **`Pin`** | Fixes a value in memory — required because async state machines can reference themselves. |
| **`Waker`** | Tells the executor "poll me again, I have data now." |
| **Desugaring** | `async fn` → state machine enum → `Future::poll()` implementation. |
| **Zero allocations** | The state machine is a fixed-size enum on the stack (or heap if boxed). |
