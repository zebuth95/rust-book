# Defining Modules to Control Scope and Privacy
- **Modules** help to organize the code with a crate by grouping related functionality together.
They also control the **privacy** of the items, meaning which parts of your code are accessible to the external code.
- The **mod** keyword is used to declare a module. Modules can be nested within each other.
- By default, all item (functions, structs, enums etc.) are private within the module.
To make an item public and accessible from a parent module, you must use the **pub** keyword.


### Example:
This code shows a garden module containing a vegetables module, which holds a public Asparagus struct.

    // In src/lib.rs
    pub mod garden {
        pub mod vegetables {
            pub struct Asparagus {
                pub name: String,
            }
        }
    }

In the example above, the Asparagus struct and its name field are made public with the pub keyword so they can be used outside their modules.
