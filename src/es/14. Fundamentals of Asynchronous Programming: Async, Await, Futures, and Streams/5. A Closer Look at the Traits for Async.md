# Una Mirada Más Cercana a los Traits para Async

Todo el sistema asíncrono está construido sobre unos pocos traits clave.

## El Trait `Future`

```rust
use std::pin::Pin;
use std::task::{Context, Poll};

trait Future {
    type Output;

    fn poll(self: Pin<&mut Self>, cx: &mut Context<'_>) -> Poll<Self::Output>;
}
```

| Parte | Propósito |
|-------|---------|
| **`Output`** | El tipo producido cuando el future se completa. |
| **`Pin<&mut Self>`** | Evita que el future sea movido en memoria — crítico para máquinas de estado autoreferenciales. |
| **`Context` / `Waker`** | El future registra un waker para que el executor sepa que debe hacer poll nuevamente cuando los datos estén listos. |
| **`Poll::Ready(T)`** | Computación terminada con el valor `T`. |
| **`Poll::Pending`** | Aún no está listo — llama a `poll()` de nuevo cuando sea despertado. |

## Cómo se Desglosa `async`

El compilador transforma `async fn` en un struct + implementación de `Future`:

```rust
// This:
async fn example() -> u32 {
    let x = 1;
    let y = 2;
    x + y
}

// Roughly becomes:
enum ExampleStateMachine {
    Start,
    Done(u32),
}

impl Future for ExampleStateMachine {
    type Output = u32;
    fn poll(self: Pin<&mut Self>, _cx: &mut Context<'_>) -> Poll<u32> {
        match *self {
            ExampleStateMachine::Start => {
                let x = 1;
                let y = 2;
                let result = x + y;
                *self = ExampleStateMachine::Done(result);
                Poll::Ready(result)
            }
            ExampleStateMachine::Done(result) => Poll::Ready(result),
        }
    }
}
```

## El Trait `Stream`

```rust
trait Stream {
    type Item;
    fn poll_next(self: Pin<&mut Self>, cx: &mut Context<'_>) -> Poll<Option<Self::Item>>;
}
```

- Mismo patrón que `Future` pero produce **múltiples elementos** a lo largo del tiempo.
- `Poll::Ready(Some(item))` — siguiente valor disponible.
- `Poll::Ready(None)` — stream agotado.
- `Poll::Pending` — esperando el siguiente valor.

---

## Puntos Clave

| Concepto | Significado |
|---------|---------|
| **`Pin`** | Fija un valor en memoria — necesario porque las máquinas de estado async pueden referenciarse a sí mismas. |
| **`Waker`** | Le dice al executor "hazme poll de nuevo, ahora tengo datos." |
| **Desglose** | `async fn` → enum de máquina de estado → implementación de `Future::poll()`. |
| **Cero asignaciones** | La máquina de estado es un enum de tamaño fijo en la pila (o en el heap si está boxed). |
