# Generic Types, Traits, and Lifetimes

Rust's tools for writing flexible, reusable, safe code — all at compile time, zero runtime cost.

| Concept | Purpose |
|---------|---------|
| **Generics** | Abstract type placeholders. Monomorphized at compile time → no overhead. |
| **Traits** | Define shared behavior across types (like interfaces). |
| **Lifetimes** | Ensure references stay valid. Prevent dangling pointers. |

---

## Combined Example

```rust
fn longest_with_announcement<'a, T>(x: &'a str, y: &'a str, ann: T) -> &'a str
where
    T: std::fmt::Display,
{
    println!("Announcement! {ann}");
    if x.len() > y.len() { x } else { y }
}
```

## Key Points

- **Zero-cost abstractions** — generics compile to concrete code per type (monomorphization).
- **Compile-time polymorphism** — no runtime dispatch unless you opt in with `dyn Trait`.
- **Lifetimes are generics too** — they're type-level annotations for reference safety.
