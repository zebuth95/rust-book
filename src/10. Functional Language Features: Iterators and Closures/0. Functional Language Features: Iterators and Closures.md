# Functional Language Features: Iterators and Closures

Rust's design has taken inspiration from many existing languages and techniques, and one significant influence is
functional programing.

- **Closures** a function-like construct you can store in a variable (anonymous functions with capture capabilities).
- **Iterators** a way of processing a series of elements (a lazy, zero-cost abstraction for sequential data processing).
- The performance of closures and iterator

We’ve already covered some other Rust features, such as pattern matching and enums, that are also influenced by the functional style. Because mastering closures and iterators is an important part of writing idiomatic, fast Rust code, we’ll devote this entire chapter to them.
