# State Pattern with Trait Objects

A common OOP pattern — change behavior by swapping internal state.

```rust
trait State {
    fn request_review(self: Box<Self>) -> Box<dyn State>;
    fn approve(self: Box<Self>) -> Box<dyn State>;
    fn content<'a>(&self, post: &'a Post) -> &'a str { "" }
}

struct Draft {}
impl State for Draft { /* ... */ }

struct Published {}
impl State for Published {
    fn content<'a>(&self, post: &'a Post) -> &'a str { &post.content }
}

pub struct Post {
    state: Option<Box<dyn State>>,
    content: String,
}
```

> **Note:** Each state is a concrete type implementing the `State` trait. The `Post` delegates behavior to its current state.
