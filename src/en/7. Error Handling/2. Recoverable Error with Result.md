# `Result<T, E>` — Recoverable Errors

```rust
enum Result<T, E> {
    Ok(T),
    Err(E),
}
```

You **must** handle both variants.

## Handling with `match`

```rust
use std::fs::File;

let f = File::open("hello.txt");
let f = match f {
    Ok(file) => file,
    Err(err) => panic!("{err}"),
};
```

## `unwrap()` and `expect()`

Shortcuts — panic on error.

```rust
let f = File::open("hello.txt").unwrap();
let f = File::open("hello.txt").expect("Failed to open");
```

## `?` operator — propagate errors

```rust
use std::fs::File;
use std::io::{self, Read};

fn read_username() -> Result<String, io::Error> {
    let mut s = String::new();
    File::open("hello.txt")?.read_to_string(&mut s)?;
    Ok(s)
}
```

- If `Ok` → unwrap the value and continue.
- If `Err` → return it early from the function.
- Only works in functions that return `Result` or `Option`.

> **Key point:** Use `?` to propagate errors. Use `unwrap()` only in tests/prototypes.

#### Custom Error Types with `thiserror`

```rust
use std::fs::File;
use std::io::{self, Read};
use thiserror::Error;

#[derive(Error, Debug)]
pub enum MyError {
    #[error("I/O error: {0}")]
    Io(#[from] io::Error),
    #[error("parse error: {0}")]
    Parse(String),
}

fn read_and_parse() -> Result<i32, MyError> {
    let mut s = String::new();
    File::open("numbers.txt")?.read_to_string(&mut s)?;
    s.trim().parse::<i32>().map_err(|e| MyError::Parse(e.to_string()))
}
```

#### Converting Errors with `.map_err()`

```rust
fn read_number() -> Result<i32, String> {
    let content = std::fs::read_to_string("num.txt")
        .map_err(|e| format!("File error: {e}"))?;
    content.trim().parse::<i32>()
        .map_err(|e| format!("Parse error: {e}"))
}
```

#### Box<dyn Error> — Simple Dynamic Errors

```rust
fn flexible_fn() -> Result<i32, Box<dyn std::error::Error>> {
    let f = std::fs::read_to_string("file.txt")?;   // io::Error → Box<dyn Error>
    let n = f.trim().parse::<i32>()?;                 // ParseIntError → Box<dyn Error>
    Ok(n)
}
```
