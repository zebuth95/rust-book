# Tipos de Datos

Cada valor tiene un tipo. Dos categorías: **escalares** y **compuestos**.

## Tipos Escalares

| Tipo | Ejemplos |
|---|---|
| Enteros | `i8`, `u8`, `i32` (por defecto), `u64`, `isize`, `usize` |
| Flotantes | `f32`, `f64` (por defecto) |
| Booleano | `true`, `false` |
| Carácter | `'a'`, `'😻'` (4 bytes, Unicode) |

```rust
let a: i32 = -10;      // signed
let b: u8 = 255;       // unsigned
let c = 2.0;           // f64
let d: f32 = 3.0;      // f32
let t = true;
let heart = '😻';
```

## Enteros con Signo vs sin Signo

| | Con signo (`i`) | Sin signo (`u`) |
|---|---|---|
| **Rango** | Incluye negativos | Solo cero y positivos |
| **Ejemplo** | `i8`: **-128** a **127** | `u8`: **0** a **255** |
| **Nomenclatura** | `i` + bits (`i8`, `i16`, `i32`, `i64`, `i128`, `isize`) | `u` + bits (`u8`, `u16`, `u32`, `u64`, `u128`, `usize`) |
| **Cuándo usarlos** | Valores pueden ser negativos (temperatura, saldo) | Valores siempre no negativos (conteos, IDs, tamaños) |

- `isize` / `usize` dependen de la arquitectura de tu CPU (32 o 64 bits).
- **El tipo entero por defecto** es `i32` — buen balance entre velocidad y rango.
- El comportamiento de desbordamiento difiere: en modo debug **entra en pánico**, en modo release **se envuelve** (complemento a dos).

```rust
let signed: i8 = -128;        // min i8
let unsigned: u8 = 255;       // max u8
// let oops: u8 = -5;         // compile error: can't assign negative to unsigned

let arch_size: usize = 8;     // 8 on 64-bit machines
```

## Tipos Compuestos

### Tupla — longitud fija, tipos mixtos

```rust
let tup: (i32, f64, u8) = (500, 6.4, 1);
let (x, y, z) = tup;          // destructuring
let first = tup.0;             // index access
```

### Arreglo — longitud fija, mismo tipo

```rust
let arr: [i32; 3] = [1, 2, 3];
let first = arr[0];   // index
```

> **Nota:** Usa `Vec` (asignado en heap) para arreglos dinámicos.

## El Tipo Unidad `()`

La tupla vacía `()` se llama el **tipo unidad**. Tiene exactamente un valor — también `()`.

- Se usa como tipo de retorno para funciones que no devuelven un valor significativo.
- `fn foo() {}` implícitamente retorna `()`.
- Las expresiones que terminan con `;` evalúan a `()`.

```rust
fn greet() {                        // returns ()
    println!("hello");
}

let result: () = greet();
```

## El Tipo Nunca `!`

El tipo nunca `!` representa cómputos que **nunca producen un valor**.

- Funciones que nunca retornan: `fn forever() -> ! { loop {} }`.
- `panic!()`, `continue`, `return` tienen tipo `!`.
- Se usa para compatibilidad de tipos: `!` se coerce a cualquier tipo.

```rust
let x = if true { 5 } else { panic!() };  // x is i32, because ! coerces
```

## Conversión Explícita de Tipos (`as`)

Usa la palabra clave `as` para convertir entre tipos numéricos intencionalmente.

```rust
let a = 3.14_f64;
let b = a as i32;     // 3, truncates

let c = 255_u8;
let d = c as i32;     // 255, safe widening

let e = 300_u16;
let f = e as u8;      // 44, wraps around (300 - 256 = 44)
```

> **Precaución:** `as` truncará o envolverá silenciosamente cuando el tipo destino no pueda contener el valor.
