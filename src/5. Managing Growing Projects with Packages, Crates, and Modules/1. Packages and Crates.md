# Packages and Crates
- **Packages** are units of code managed by Cargo. A package contains one or more **crates**.
A package can have a **Cargo.toml** file that describes hot to build  the contained crates.
- **Crates** are the smallest unit of code that Rust compiler consider at a time.
They can be either a **library create** (a reusable collection of functions and data).
- **Note:** A package must contains at least one crate. It can have zero or one library crates and any number of binary crates.


### Example:
When you run cargo new my_project, Cargo creates a new package.

    .
    ├── Cargo.toml
    └── src
    └── main.rs

In this structure, my_project is the package, and src/main.rs is the binary crate. If you were to add src/lib.rs, that would be a separate library crate within the same package.
