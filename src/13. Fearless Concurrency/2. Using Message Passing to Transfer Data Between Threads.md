# Using Message Passing to Transfer Data Between Threads
This is the "communication sequential process" (**CSP**) model, often know  as "Do not communicate by sharing memory; instead, share memory by communicating."
Data is sent between threads using channels.

## Implementation:
Rut's standard library provides Multiples Producer, Single Consumer (**MPSC**) channels via the **std::sync::mpsc** module.

- mpsc::channel() returns a pair: a Transmitter (tx) and a Receiver (rx).
- tx.send(data) transfers ownership of data to the channel.
- rx.recv() (blocking) or rx.try_recv() (non-blocking) retrieves the data.

## Safety:
The ownership system ensure that once data is sent, the sender cannot use it anymore, preventing a data race condition.
Cloning a Transmitter (tx.clone()) allows multiple threads to send data to the same receiver.

### Example (Sending and Receiving):

    use std::sync::mpsc;
    use std::thread;
    
    let (tx, rx) = mpsc::channel();
    
    thread::spawn(move || {
        let val = String::from("Hello from thread!");
        tx.send(val).unwrap();
        // The 'val' variable cannot be used here after the send() call (ownership was moved)
    });
    
    // The main thread waits for and receives the value
    let received = rx.recv().unwrap();
    println!("Got: {}", received);
