# Traits Avanzados

## Tipos Asociados

```rust
pub trait Iterator {
    type Item;
    fn next(&mut self) -> Option<Self::Item>;
}
```

## Sobrecarga de Operadores

```rust
use std::ops::Add;

#[derive(Debug, PartialEq)]
struct Point { x: i32, y: i32 }

impl Add for Point {
    type Output = Point;
    fn add(self, other: Point) -> Point {
        Point { x: self.x + other.x, y: self.y + other.y }
    }
}
```

## Supertraits

```rust
trait Animal { fn name(&self) -> &str; }
trait Canine: Animal {  // Canine requires Animal
    fn bark(&self) { println!("{} says woof!", self.name()); }
}
```

## Sintaxis Completamente Cualificada

```rust
trait Pilot { fn fly(&self); }
trait Wizard { fn fly(&self); }
struct Human;
impl Human { fn fly(&self) { println!("waving arms"); } }

let person = Human;
Pilot::fly(&person);   // call trait method explicitly
Wizard::fly(&person);
```

> **Nota:** Usa `Trait::method(&self)` cuando hay un conflicto de nombres.

#### Generic Associated Types (GATs)

Los GATs permiten que los tipos asociados tengan sus propios parámetros genéricos. Estabilizados en Rust 1.65.

```rust
trait Iterable {
    type Item<'a>;
    type Iter<'a>: Iterator<Item = Self::Item<'a>> where Self: 'a;

    fn iter<'a>(&'a self) -> Self::Iter<'a>;
}

impl<T: 'static> Iterable for Vec<T> {
    type Item<'a> = &'a T;
    type Iter<'a> = std::slice::Iter<'a, T>;

    fn iter<'a>(&'a self) -> Self::Iter<'a> {
        self.as_slice().iter()
    }
}
```

Casos de uso:
- Lending iterators (el iterador produce referencias vinculadas al préstamo).
- Traits asíncronos con futures asociados.

> **Nota:** Los GATs permiten patrones expresivos como lending iterators y traits asíncronos que anteriormente eran imposibles sin boxing.
