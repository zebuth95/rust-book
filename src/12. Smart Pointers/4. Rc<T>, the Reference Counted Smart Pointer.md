# Rc<T>, the Reference Counted Smart Pointer
**Rc<T>** Reference Counting allow a value to have **multiple owners**. It's used in single-thread scenarios where
multiple parts of a program need to read the same data and own it.

## Mechanism:
It keeps a count of how many active references there are. The data is only cleaned up when the reference count reaches zero.

## Immutability:
Data wrapped in Rc<T> is immutable. To achieve interior mutability, it must be paired with RefCell<T>.

### Example (Shared Ownership):

    use std::rc::Rc;
    
    let a = Rc::new(String::from("shared data"));
    println!("Count after creating a: {}", Rc::strong_count(&a)); // Count is 1
    
    let b = Rc::clone(&a); // Increments the count but performs a cheap pointer copy
    println!("Count after creating b: {}", Rc::strong_count(&a)); // Count is 2
    
    // 'b' is dropped here, count becomes 1
    // 'a' is dropped here, count becomes 0, data is cleaned up
