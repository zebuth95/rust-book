# Comments
All programmers strive (esforzar) to amke their code easy to understand.

## Simple Line Comment
    // hello, world

## Block Comments

    /*
    This is a multi-line block comment.
    It can explain a complex section of code
    or temporarily disable a large block.
    */
    fn example_function() {
        /*
        This is a nested block comment
        within the function.
        */
        println!("Hello from example_function!");
    }

## Documentation Comments
- Doc comments are a special type of comment used for generating documentation with rustdoc.
- They start with /// for items (functions, structs, enums, etc.) and //! for modules or crates.
- Doc comments can contain Markdown formatting and are essential for creating well-documented Rust libraries.
