# Extending Cargo with Custom Commands
Cargo is extensible. Any executable file in your PATH named cargo-foo can be run as a Cargo subcommand using cargo foo. This allows you to integrate custom tools or third-party scripts seamlessly into the Cargo workflow.

### Community Integration:
Many popular Rust tools, like cargo clippy (a linter) and cargo audit (a vulnerability checker), are simply external binaries that follow this naming convention.


### Example:
If you have a binary named ~/.cargo/bin/cargo-watch, you can run it as:

    cargo watch
