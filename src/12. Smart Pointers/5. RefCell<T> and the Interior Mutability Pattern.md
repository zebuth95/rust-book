# RefCell<T> and the Interior Mutability Pattern
**RefCell<T>** the **Interior Mutability Pattern** allows you to mutate data even when you have an immutable reference to it.

## Mechanism:
Instead of checking Rust's borrowing rules at compile time, **RefCell<T>** move the check to runtime.

- **borrow()**: Provides an immutable reference **&T**. Multiple immutable borrows are allowed.
- **borrow_mut()**: Provides a mutable reference **&mut T**. Only one mutable borrow is allowed.

## Common Use:
Used with Rc<T> to allow mutable access to shared data in single-threaded scenarios: Rc<RefCell<T>>.

    use std::cell::RefCell;
    
    let x = RefCell::new(String::from("Hello"));
    let mut s = x.borrow_mut(); // Creates a mutable borrow
    s.push_str(" World");
    // s goes out of scope here, releasing the mutable borrow
    
    // If another mutable borrow was attempted before 's' went out of scope, it would panic.
    println!("{}", x.borrow()); // Output: Hello World
