# Validando Referencias con Tiempos de Vida

Los tiempos de vida aseguran que las referencias sigan siendo válidas mientras sea necesario — previenen **punteros colgantes** en tiempo de compilación.

## El Problema: Referencia Colgante

```rust
fn main() {
    let r;

    {
        let x = 5;
        r = &x; // error: `x` does not live long enough
    }

    println!("r: {r}"); // r would point to freed memory
}
```

El compilador detecta esto en tiempo de compilación — no se necesita un panic en ejecución.

## Sintaxis de Anotación de Tiempo de Vida

Las anotaciones de tiempo de vida **describen relaciones** entre referencias. No cambian los tiempos de vida.

```rust
// `'a` means: returned ref lives as long as the shortest of the two inputs
fn longest<'a>(x: &'a str, y: &'a str) -> &'a str {
    if x.len() > y.len() { x } else { y }
}
```

- `&'a str` = "un slice de string que vive durante el tiempo de vida `'a`"
- El tipo de retorno también debe ser `&'a str` porque podría ser `x` o `y`.

## Tiempo de Vida en Structs

```rust
struct ImportantExcerpt<'a> {
    part: &'a str,
}

let novel = String::from("Call me Ishmael. Some years ago...");
let first_sentence = novel.split('.').next().expect("Could not find a '.'");
let i = ImportantExcerpt { part: first_sentence };
// `i` cannot outlive `novel` because `part` borrows from it
```

## Reglas de Elisión de Tiempo de Vida

Rust agrega **implícitamente** tiempos de vida en patrones comunes — no es necesario escribirlos manualmente.

| Regla | Qué significa |
|------|---------------|
| **Cada referencia de entrada** tiene su propio tiempo de vida | `fn foo(x: &str)` → `fn foo<'a>(x: &'a str)` |
| **Una referencia de entrada** → la salida obtiene ese tiempo de vida | `fn foo(x: &str) -> &str` → `fn foo<'a>(x: &'a str) -> &'a str` |
| **Entrada `&self`** → la salida obtiene el tiempo de vida de `&self` | `fn get(&self) -> &str` → `fn get<'a>(&'a self) -> &'a str` |

## Tiempo de Vida Estático

```rust
let s: &'static str = "I live forever in the binary";
```

- `'static` = la referencia vive durante todo el programa.
- Los literales de string son `&'static str` por defecto.
- Úsalo con moderación — la mayoría de las veces querrás tiempos de vida regulares.

---

## Puntos Clave

| Concepto | Significado |
|---------|---------|
| **Tiempo de vida `'a`** | Etiqueta el ámbito de una referencia; el compilador verifica su validez. |
| **Puntero colgante** | Referencia a memoria desasignada — prevenido por los tiempos de vida. |
| **Elisión** | El compilador completa los tiempos de vida automáticamente para patrones comunes. |
| **`'static`** | La referencia vive durante toda la duración del programa. |
