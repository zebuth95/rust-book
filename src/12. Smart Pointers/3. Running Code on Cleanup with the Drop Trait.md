# Running Code on Cleanup with the Drop Trait
The second trait to the smart pointer patter it **Drop**, which lets you customize what happens when a value is about to go out oof scope.
You can provide an implementation for the **Drop** trait on any type, and that code can be used to release resources like
file or network connections. This is where memory cleanup, file closing, or other resource release logic occurs.

## Mechanism:
The compiler automatically inserts a call to the drop method in the Drop trait implementation just before the value's scope ends.

## Manual Drop:
You cannot manually call the drop method directly (it's prevented by the compiler). To force an early cleanup, you must use the std::mem::drop function.

### Example (The drop method runs automatically):

    struct CustomSmartPointer {
        data: String,
    }
    
    // Implementing the Drop trait
    impl Drop for CustomSmartPointer {
        fn drop(&mut self) {
            println!("Dropping CustomSmartPointer with data `{}`!", self.data);
        }
    }
    
    fn main() {
        let c = CustomSmartPointer { data: String::from("resource") };
        // The println! will execute when 'c' goes out of scope.
        // To force it early: std::mem::drop(c);
    }
