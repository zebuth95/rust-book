# Characteristics of Object-Oriented Languages
Traditional OOP languages are often defined by three characteristics:

- **Encapsulation**: Packaging data adn methods that operate on that data into a single unit. In Rust, this is achieved 
through **structs** and the **pub** keyword for controlling visibility.
- **Inheritance**: A mechanism fro one object to inherit that characteristics of another. 
Rust **does not have inheritance**; it favors **composition** and **traits** instead.
- **Polymorphism**: The ability of objects to take on different forms or for code to work with values of different types.
Rust achieves this primarily through **Generics** and **Trait Objects**.


# Example (Encapsulation):

    pub struct AveragingCollection {
        // Data is private, only accessible via public methods
        list: Vec<i32>,
        average: f64,
    }
    
    impl AveragingCollection {
        // Public method to modify the private data
        pub fn add(&mut self, value: i32) {
            self.list.push(value);
            // Internal logic is encapsulated
            self.update_average();
        }
        
        fn update_average(&mut self) {
            // ... calculation logic ...
            self.average = self.list.iter().sum::<i32>() as f64 / self.list.len() as f64;
        }

        // Public method to read the private data
        pub fn average(&self) -> f64 {
            self.average
        }
    }

# (Inheritance - Rust's approach: Composition and Traits)
Instead of using class inheritance, Rust promotes composition (where one struct contains another) and traits (defining shared behavior).

    // Trait defining a shared behavior
    pub trait Summary {
        fn summarize(&self) -> String;
    }
    
    // Struct representing a News Article
    pub struct NewsArticle {
        pub headline: String,
        pub location: String,
        // Note: No "base class" inheritance
    }
    
    // Struct representing a Tweet
    pub struct Tweet {
        pub username: String,
        // Instead of inheriting from NewsArticle, we define behavior
    }
    
    // Implement the Summary trait for NewsArticle
    impl Summary for NewsArticle {
        fn summarize(&self) -> String {
            format!("{}, by ({})", self.headline, self.location)
        }
    }
    
    // Implement the Summary trait for Tweet
    impl Summary for Tweet {
        fn summarize(&self) -> String {
            format!("@{}: (Tweet content not shown)", self.username)
        }
    }
    
    // The use of traits allows different types to share a capability
    fn notify(item: &impl Summary) {
        println!("Breaking news! {}", item.summarize());
    }

#### Explanation:
NewsArticle and Tweet don't inherit from a common base type, but they share the summarize behavior by implementing the Summary trait. This is Rust's way of achieving code reuse and polymorphism without traditional inheritance.


## (Polymorphism - Generics and Trait Objects)
Polymorphism in Rust is achieved primarily through Generics (for monomorphization, or static dispatch) and Trait Objects (for dynamic dispatch).


    // Generic function that can work with any type T, as long as T implements the Display trait
    // This is monomorphized at compile time, creating specific function versions for each type used.
    fn print_item<T: std::fmt::Display>(item: T) {
        println!("The item is: {}", item);
    }

    pub fn demo_generics() {
        print_item(5);         // T is inferred as i32
        print_item("hello");   // T is inferred as &str
        print_item(3.14);      // T is inferred as f64
    }

### Trait Object Example (Dynamic Polymorphism):

    // Trait defining a common interface
    pub trait Draw {
        fn draw(&self);
    }
    
    // Concrete types that implement the trait
    pub struct Button {
        pub width: u32,
    }
    
    impl Draw for Button {
        fn draw(&self) {
            println!("Drawing a button of width {}", self.width);
        }
    }
    
    pub struct SelectBox {
        pub options: Vec<String>,
    }
    
    impl Draw for SelectBox {
        fn draw(&self) {
            println!("Drawing a select box with {} options", self.options.len());
        }
    }
    
    // A vector holding different types (Button and SelectBox) that all implement Draw.
    // This is achieved by using a 'Trait Object': Box<dyn Draw>.
    pub fn demo_trait_objects() {
        let screen_components: Vec<Box<dyn Draw>> = vec![
        Box::new(Button { width: 50 }),
        Box::new(SelectBox { options: vec!["A".to_string(), "B".to_string()] }),
        ];

        for component in screen_components {
            // Dynamic dispatch: the correct 'draw' method is chosen at runtime.
            component.draw();
        }
    }

#### Explanation:
The Vec<Box<dyn Draw>> allows a collection of different types (Button and SelectBox) to be treated uniformly as a Draw 
object. The draw() method to call is determined at runtime, which is the essence of dynamic polymorphism.
