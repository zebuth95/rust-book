# Smart Pointers

Smart pointers own data and act like references (via `Deref`) with cleanup logic (via `Drop`).

| Pointer | Description |
|---|---|
| `Box<T>` | Heap allocation |
| `Rc<T>` | Reference-counted (single-thread, shared ownership) |
| `RefCell<T>` | Runtime borrow checking (interior mutability) |
| `Weak<T>` | Non-owning reference, prevents cycles |
| `Cow<T>` | Clone-on-write — borrows or owns data lazily |

## `Cow<T>` (Clone-on-Write)

An enum that either borrows (`Cow::Borrowed`) or owns (`Cow::Owned`) data. Clones only when mutation is needed.

```rust
use std::borrow::Cow;

fn normalize(name: &str) -> Cow<str> {
    if name.contains(' ') {
        // Needs modification — allocate owned String
        name.replace(' ', "_").into()
    } else {
        // No modification needed — return borrowed
        Cow::Borrowed(name)
    }
}

let simple = normalize("hello");      // Cow::Borrowed, no allocation
let spaced = normalize("hello world"); // Cow::Owned, allocated once
```

- **Use for**: Functions that sometimes modify input, sometimes return as-is (zero-cost).
- **Key methods**: `.to_mut()`, `.into_owned()`.

## `Pin<T>` — Pinning

`Pin<Box<T>>` or `Pin<&mut T>` guarantees the value **will not move in memory** after being pinned.

```rust
use std::pin::Pin;

let data = String::from("hello");
let pinned: Pin<Box<String>> = Box::pin(data);
// pinned.as_ref().get_ref() — access immutably
```

- **Required** for self-referential structs (async state machines).
- `Pin<&mut T>` is the type received in `Future::poll()`.
