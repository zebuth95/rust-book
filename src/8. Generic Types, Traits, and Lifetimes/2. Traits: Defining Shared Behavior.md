# Traits: Defining Shared Behavior

Traits = Rust's version of **interfaces**. Define a set of methods that types can implement.

## Defining a Trait

```rust
pub trait Summary {
    fn summarize(&self) -> String;                           // no default
    fn summarize_author(&self) -> String {
        format!("(Read more from {})", self.summarize())      // default impl
    }
}
```

## Implementing a Trait

```rust
pub struct NewsArticle {
    pub headline: String,
    pub author: String,
    pub location: String,
}

impl Summary for NewsArticle {
    fn summarize(&self) -> String {
        format!("{}, by {} ({})", self.headline, self.author, self.location)
    }
}

pub struct Tweet {
    pub username: String,
    pub content: String,
}

impl Summary for Tweet {
    fn summarize(&self) -> String {
        format!("{}: {}", self.username, self.content)
    }
}
```

## Traits as Parameters

```rust
// `impl Trait` syntax — accepts any type implementing `Summary`
pub fn notify(item: &impl Summary) {
    println!("Breaking news! {}", item.summarize());
}
```

### Trait Bound Syntax (equivalent)

```rust
pub fn notify<T: Summary>(item: &T) {
    println!("Breaking news! {}", item.summarize());
}
```

### Multiple Trait Bounds

```rust
// Must implement BOTH Summary AND Display
pub fn notify(item: &(impl Summary + Display)) {}

// Same with generic syntax
pub fn notify<T: Summary + Display>(item: &T) {}

// Cleaner with `where` for complex bounds
pub fn notify<T, U>(item: &T) -> String
where
    T: Summary + Display,
    U: Clone + Debug,
{}
```

## Returning `impl Trait`

```rust
fn returns_summarizable() -> impl Summary {
    Tweet {
        username: String::from("horse_ebooks"),
        content: String::from("people"),
    }
}
```

> **Note:** Can only return **one concrete type** with `impl Trait`. For multiple types, use `Box<dyn Trait>`.

---

## Key Points

| Concept | Meaning |
|---------|---------|
| **Default impl** | Methods can have default bodies. Implementors can override. |
| **Trait bound** | `T: TraitName` — restricts a generic to types implementing the trait. |
| **`impl Trait`** | Shorthand for `T: Trait` in function parameters. |
| **`+` syntax** | Combine multiple bounds: `T: TraitA + TraitB`. |
| **`where` clause** | Cleaner syntax for complex trait bounds. |
