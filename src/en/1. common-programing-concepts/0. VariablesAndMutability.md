# Variables and Mutability

Variables are **immutable** by default.

## `let` (immutable)

```rust
let x = 5;
// x = 6;  // ❌ compile error: cannot assign twice
```

## `let mut` (mutable)

```rust
let mut x = 5;
x = 6;  // ✅ allowed
```

## `const` (always immutable)

```rust
const MAX_POINTS: u32 = 100_000;
```

## Shadowing

Re-declare with `let` to create a new variable (can change type).

```rust
let x = 5;
let x = x + 1;       // shadows, x = 6
let x = "hello";     // shadows with different type
```

> **Key point:** Shadowing is NOT mutation. The old value is replaced.

## `static` — Global Variables

A `static` variable has a **fixed memory address** and lives for the entire program.

```rust
static MAX_CONNECTIONS: u32 = 1024;
static mut COUNTER: u32 = 0;  // unsafe to read/write
```

- `static` variables are **not inlined** at compile time.
- Mutable statics (`static mut`) are inherently unsafe and require `unsafe` blocks to read or write.

## `const` vs `static`

| | `const` | `static` |
|---|---|---|
| Inlined at compile time | Yes | No (fixed address) |
| Has memory address | No | Yes |
| Unsafe mutation | N/A | Allowed with `static mut` |

## Scope and Shadowing (In Depth)

Shadowing lets you reuse a variable name within a scope by redeclaring it with `let`. The new variable **hides** the previous one.

### Shadowing in nested scopes

```rust
let x = 5;
{
    let x = 99;       // shadows outer x inside this block
    println!("{x}");  // 99
}
println!("{x}");      // 5 — outer x is unaffected
```

### Shadowing vs `mut`

Shadowing allows you to **change the type** of a variable; `mut` does not.

```rust
let x = "hello";
let x = x.len();  // ✅ shadowing: &str → usize

let mut y = "hello";
// y = y.len();   // ❌ error: expected &str, found usize (mut does NOT allow type change)
```

> **Key point:** Use shadowing when you want to transform a value while keeping the same name. Use `mut` when you need to reassign the same type repeatedly.
