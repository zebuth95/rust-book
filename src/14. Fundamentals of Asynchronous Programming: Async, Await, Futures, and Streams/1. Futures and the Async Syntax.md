# Futures and the Async Syntax
**Future** is a **trait** representing an asynchronous computation that may not have completed yet. 
it's an enum that returns either:

- Poll::Ready(T): The result is available.
- Poll::Pending: The computation is not ready, and the executor should try again later.

### async Keyword:
Used to define a function that implicitly return a **Future**.
The function's body is the code that will be executed asynchronously when the **Future** is polled.

## await Keyword:
Used inside an **async** function to pause execution until Future it;s called on resolves (return **Poll:Ready**).
it yields control back to the executor, allowing other tasks to run.

    // The 'async' keyword transforms the function body into a Future.
    async fn fetch_data() -> String {
        // Simulate an asynchronous operation that takes time
        tokio::time::sleep(std::time::Duration::from_secs(1)).await;
        "Data received".to_string()
    }
    
    async fn main_async() {
        println!("Starting fetch...");
        // 'await' pauses the execution of main_async until fetch_data completes.
        let result = fetch_data().await;
        println!("{}", result);
    }
    // Note: The async function itself must be run by an executor (e.g., Tokio or async-std).
