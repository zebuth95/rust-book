# Extensible Concurrency with the Send and Sync Traits
**Send** and **Sync** are two marker traits that enable Rust's compile-time concurrency safety. Most types implement
them automatically, but they exits to prevent unsafe data transfers.

# Send Trait:
Allows Ownership to a type to be transferred across thread boundaries. Most primitive types (like numbers) and common 
smart pointers (Box<T>, Arc<T>) implement Send. Rc<T> does not implement Send because transferring it would violate its single threaded guarantee.

## Sync Trait:
Allows a type to be **safely referenced** across thread boundaries. it a type T is Sync, then reference to it (&T) is Send.
This way Mutex<T> is Sync: a shared, immutable reference to the mutex is safe to send between threads.

## Safety Rule:
Types composed entirely of Send types are automatically Send. Similarly, for Sync.
The standard library types that don't implement these traits (like Rc<T> and RefCell<T>) are the ones that could 
introduce data races if shared carelessly. Rust forces you to use the thread-safe alternatives (Arc<T> and Mutex<T>) 
instead.
