# Working With Any Number of Futures

Two key macros for managing multiple futures.

## `join!` — Wait for **All** Futures

```rust
use tokio::time::{sleep, Duration};

async fn fetch(id: u32) -> String {
    sleep(Duration::from_millis(id as u64 * 50)).await;
    format!("result {id}")
}

#[tokio::main]
async fn main() {
    let (a, b, c) = tokio::join!(fetch(1), fetch(2), fetch(3));
    println!("{a}, {b}, {c}"); // All done in ~150ms, not 300ms
}
```

## `select!` — Wait for the **First** Future

```rust
use tokio::time::{sleep, Duration};

async fn fast() -> &'static str {
    sleep(Duration::from_millis(100)).await;
    "Fast Task Completed"
}

async fn slow() -> &'static str {
    sleep(Duration::from_millis(500)).await;
    "Slow Task Completed"
}

#[tokio::main]
async fn main() {
    let winner = tokio::select! {
        result = fast() => format!("Winner: {result}"),
        result = slow() => format!("Winner: {result}"),
    };
    println!("{winner}"); // "Winner: Fast Task Completed" (~100ms)
}
```

## Comparison

| Macro | Behavior | Use Case |
|-------|----------|----------|
| **`join!`** | Runs all, waits for **all** to finish | Fetch multiple independent resources |
| **`select!`** | Runs all, returns **first** to complete | Timeouts, races, fallbacks |

> **Note:** `select!` drops the other futures when one wins. The dropped futures are still polled — it's a race, not a cancellation. Use `tokio::select!(biased; ...)` for priority-based selection.

#### `FuturesUnordered` — Dynamic Set of Futures

From the `futures` crate — poll many futures concurrently and get results as they complete.

```rust
use futures::stream::FuturesUnordered;
use futures::StreamExt;

let mut futures = FuturesUnordered::new();

for i in 0..10 {
    futures.push(async move {
        tokio::time::sleep(std::time::Duration::from_millis(i * 20)).await;
        i
    });
}

// Process results as they complete (not in insertion order)
while let Some(result) = futures.next().await {
    println!("got: {result}");
}
```

#### `buffer_unordered` — Bounded Concurrency on a Stream

```rust
use tokio_stream::StreamExt;

async fn fetch(url: &str) -> String {
    format!("data from {url}")
}

let urls = vec!["a.com", "b.com", "c.com", "d.com"];
let results: Vec<_> = tokio_stream::iter(urls)
    .map(|url| fetch(url))
    .buffer_unordered(2)  // max 2 concurrent fetches
    .collect()
    .await;
```

This limits concurrency to N simultaneous operations while processing a stream.
