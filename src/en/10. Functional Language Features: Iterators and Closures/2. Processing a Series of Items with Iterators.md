# Iterators

Iterators are **lazy** — they do nothing until consumed.

```rust
let v1 = vec![1, 2, 3];

let iter = v1.iter();            // lazy, nothing happens
let sum: i32 = iter.sum();       // consuming adapter, drives iteration
```

## The `Iterator` Trait

```rust
pub trait Iterator {
    type Item;
    fn next(&mut self) -> Option<Self::Item>;
    // many default methods provided...
}
```

## `next()` method

```rust
let mut iter = vec![1, 2].iter();
assert_eq!(iter.next(), Some(&1));
assert_eq!(iter.next(), Some(&2));
assert_eq!(iter.next(), None);
```

## Three Forms of Iteration

| Method | Returns | Ownership |
|--------|---------|-----------|
| `.iter()` | `&T` | Borrows (read-only) |
| `.iter_mut()` | `&mut T` | Borrows (mutable) |
| `.into_iter()` | `T` | Consumes (takes ownership) |

```rust
let v = vec![1, 2, 3];

for x in &v { }          // implicit .iter()
for x in &mut v { }      // implicit .iter_mut()
for x in v { }           // implicit .into_iter(), v consumed
```

## Common Adapters (Non-Consuming)

These return a new iterator (lazy):

```rust
let v = vec![1, 2, 3, 4, 5];

let mapped: Vec<_> = v.iter().map(|x| x * 2).collect();
let filtered: Vec<_> = v.iter().filter(|&&x| x % 2 == 0).collect();
let skipped: Vec<_> = v.iter().skip(2).collect();
let taken: Vec<_> = v.iter().take(3).collect();
let folded: i32 = v.iter().fold(0, |acc, x| acc + x);
```

## Consuming Adapters

These drive the iterator to completion:

```rust
let v = vec![1, 2, 3];
let sum: i32 = v.iter().sum();
let count = v.iter().count();
let max = v.iter().max();
let min = v.iter().min();
let collected: Vec<_> = v.iter().collect();
let any = v.iter().any(|&x| x > 2);
let all = v.iter().all(|&x| x > 0);
```

## Chaining

```rust
let v = vec![3, 1, 4, 1, 5];
let result: Vec<_> = v.iter()
    .filter(|&&x| x > 2)
    .map(|x| x * 10)
    .collect();  // [30, 40, 50]
```

## `IntoIterator` Trait

Makes a type iterable with `for` loops:

```rust
pub trait IntoIterator {
    type Item;
    type IntoIter: Iterator<Item = Self::Item>;
    fn into_iter(self) -> Self::IntoIter;
}
```

Any type implementing `IntoIterator` works in `for x in value { }`.

## `FromIterator` Trait

Creates a collection from an iterator (used by `.collect()`):

```rust
pub trait FromIterator<A> {
    fn from_iter<T: IntoIterator<Item = A>>(iter: T) -> Self;
}
```

## Custom Iterator Implementation

```rust
struct Counter {
    count: u32,
}

impl Counter {
    fn new() -> Self { Counter { count: 0 } }
}

impl Iterator for Counter {
    type Item = u32;

    fn next(&mut self) -> Option<Self::Item> {
        self.count += 1;
        if self.count < 6 { Some(self.count) } else { None }
    }
}

let sum: u32 = Counter::new()
    .zip(Counter::new().skip(1))
    .map(|(a, b)| a * b)
    .filter(|x| x % 3 == 0)
    .sum();
assert_eq!(sum, 18);
```

> **Key point:** Iterators are zero-cost — they optimize to the same code as hand-written loops.
