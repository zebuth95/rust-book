# Macros

**Metaprogramming** — code that writes code.

## Declarative Macros (`macro_rules!`)

```rust
macro_rules! say_hello {
    () => { println!("Hello!"); };
    ($name:expr) => { println!("Hello, {}!", $name); };
}

say_hello!();
say_hello!("Alice");
```

## Repetition

```rust
macro_rules! my_vec {
    ($($x:expr),*) => {
        {
            let mut v = Vec::new();
            $(v.push($x);)*
            v
        }
    };
}

let v = my_vec![1, 2, 3];
```

## Procedural Macros

- `#[derive(Debug)]` — custom derive
- `#[attribute]` — attribute-like
- `custom!()` — function-like

```rust
use serde::Deserialize;

#[derive(Deserialize)]   // custom derive macro
struct Config { name: String }
```

> **Note:** Use `macro_rules!` for simple code generation. Use procedural macros for complex transformations.

#### Macro Hygiene

Macros are **hygienic** — identifiers declared inside a macro don't leak to the calling scope.

```rust
macro_rules! create_fn {
    ($name:ident) => {
        fn $name() {
            println!("called {}", stringify!($name));
        }
    };
}

create_fn!(foo);
foo(); // prints "called foo"
```

#### `$crate` — Referencing the Current Crate

Always use `$crate::` to refer to items from the macro's defining crate to avoid name resolution issues.

```rust
// In the crate where the macro is defined
macro_rules! my_error {
    ($msg:expr) => {
        $crate::MyError::new($msg)
    };
}
```
