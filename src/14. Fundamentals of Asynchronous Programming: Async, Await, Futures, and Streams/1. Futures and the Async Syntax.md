# Futures and the Async Syntax

## The `Future` Trait

A `Future` represents a value that isn't ready yet. The executor polls it repeatedly:

```rust
use std::task::Poll;

trait SimpleFuture {
    type Output;
    fn poll(&mut self) -> Poll<Self::Output>;
}
// Poll::Ready(T)  → value is available
// Poll::Pending   → not ready yet, try again later
```

## `async` / `.await`

- `async fn` — compiler transforms the function body into a **state machine** implementing `Future`.
- `.await` — yields control to the executor. The task is paused until the future is ready.

```rust
async fn fetch_data() -> String {
    // This .await suspends the function without blocking the thread
    tokio::time::sleep(std::time::Duration::from_secs(1)).await;
    "Data received".to_string()
}

async fn main_async() {
    println!("Starting fetch...");
    let result = fetch_data().await; // pause here until ready
    println!("{result}");
}
```

## What `async fn` Actually Does

```rust
// This:
async fn foo() -> u32 { 42 }

// Desugars into something like:
fn foo() -> impl Future<Output = u32> {
    async { 42 }
}
```

## `async` Blocks

You can also create futures inline without a function:

```rust
let future = async {
    println!("Hello from an async block!");
    42
};
let result = future.await;
```

---

## Key Points

| Concept | Meaning |
|---------|---------|
| **`Future`** | Trait with `poll()` → returns `Ready(T)` or `Pending`. |
| **`async fn`** | Returns `impl Future<Output = T>`. No blocking until awaited. |
| **`.await`** | Pauses the current future, yields to executor. |
| **State machine** | The compiler generates an enum that tracks progress through await points. |
| **Needs executor** | An async function does nothing until polled by a runtime like Tokio. |
