# Using Trait Objects That Allow for Values of Different Types
A **Trait Object** allows polymorphism by enabling a collection or function to accept any value whose type implements a 
specific trait, regardless of the concrete type.
This is Rust's mechanism for **dynamic dispatch**.

## Syntax:
A trait object is written as **Box<dyn Trait>** (must common) or **&dyn Trait**.
- dyn signals that this is a trait object.
- Box (or &) indicates that the value is allocated on the heap (or borrowed) because a trait object must have a guaranteed, known size at compile time.

## Mechanism (Vtable):
Trait objects use a Virtual Dispatch Table (vtable) at runtime to determine which concrete method to call, 
introducing a small runtime cost, unlike zero-cost generics (static dispatch).

### Example (Dynamic Polymorphism with Trait Objects):
Imagine a drawing application that needs a list of shapes that can all be drawn.

    // 1. Define the Trait
    pub trait Draw {
        fn draw(&self);
    }
    
    // 2. Define concrete types implementing the Trait
    pub struct Button { /* ... */ }

    impl Draw for Button {
        fn draw(&self) { println!("Drawing a Button"); }
    }
    pub struct SelectBox { /* ... */ }

    impl Draw for SelectBox {
        fn draw(&self) { println!("Drawing a SelectBox"); }
        }
    
    // 3. Use the Trait Object in a Collection
    pub struct Screen {
        // The Vec holds any type that implements the Draw trait
        pub components: Vec<Box<dyn Draw>>,
    }
    
    impl Screen {
        pub fn run(&self) {
            for component in self.components.iter() {
                // Runtime dispatch uses the vtable to call the correct draw() method
                component.draw();
            }
        }
    }
