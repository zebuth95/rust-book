# Trabajando con Cualquier Número de Futures

Dos macros clave para gestionar múltiples futures.

## `join!` — Esperar por **Todos** los Futures

```rust
use tokio::time::{sleep, Duration};

async fn fetch(id: u32) -> String {
    sleep(Duration::from_millis(id as u64 * 50)).await;
    format!("result {id}")
}

#[tokio::main]
async fn main() {
    let (a, b, c) = tokio::join!(fetch(1), fetch(2), fetch(3));
    println!("{a}, {b}, {c}"); // All done in ~150ms, not 300ms
}
```

## `select!` — Esperar por el **Primer** Future

```rust
use tokio::time::{sleep, Duration};

async fn fast() -> &'static str {
    sleep(Duration::from_millis(100)).await;
    "Fast Task Completed"
}

async fn slow() -> &'static str {
    sleep(Duration::from_millis(500)).await;
    "Slow Task Completed"
}

#[tokio::main]
async fn main() {
    let winner = tokio::select! {
        result = fast() => format!("Winner: {result}"),
        result = slow() => format!("Winner: {result}"),
    };
    println!("{winner}"); // "Winner: Fast Task Completed" (~100ms)
}
```

## Comparación

| Macro | Comportamiento | Caso de Uso |
|-------|----------|----------|
| **`join!`** | Ejecuta todos, espera a que **todos** terminen | Obtener múltiples recursos independientes |
| **`select!`** | Ejecuta todos, devuelve el **primero** en completarse | Timeouts, carreras, respaldos |

> **Nota:** `select!` descarta los otros futures cuando uno gana. Los futures descartados aún se hacen poll — es una carrera, no una cancelación. Usa `tokio::select!(biased; ...)` para selección basada en prioridad.

#### `FuturesUnordered` — Conjunto Dinámico de Futures

Del crate `futures` — hace poll a muchos futures concurrentemente y obtiene resultados a medida que se completan.

```rust
use futures::stream::FuturesUnordered;
use futures::StreamExt;

let mut futures = FuturesUnordered::new();

for i in 0..10 {
    futures.push(async move {
        tokio::time::sleep(std::time::Duration::from_millis(i * 20)).await;
        i
    });
}

// Process results as they complete (not in insertion order)
while let Some(result) = futures.next().await {
    println!("got: {result}");
}
```

#### `buffer_unordered` — Concurrencia Acotada en un Stream

```rust
use tokio_stream::StreamExt;

async fn fetch(url: &str) -> String {
    format!("data from {url}")
}

let urls = vec!["a.com", "b.com", "c.com", "d.com"];
let results: Vec<_> = tokio_stream::iter(urls)
    .map(|url| fetch(url))
    .buffer_unordered(2)  // max 2 concurrent fetches
    .collect()
    .await;
```

Esto limita la concurrencia a N operaciones simultáneas mientras se procesa un stream.

| Macro/Tipo | Comportamiento | Caso de Uso |
|-------|----------|----------|
| **`join!`** | Ejecuta todos, espera a que **todos** terminen | Obtener múltiples recursos independientes |
