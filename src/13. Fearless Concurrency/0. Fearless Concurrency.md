# Fearless Concurrency
Concurrency in Rust, often described as "Fearless Concurrency" is the approach to writing programs that can do multiple things at once without the typical bugs like
(data races or deadlocks) seen in other languages. Rust achieves this by leveraging its ownership and type system to enforce memory safety rules even across threads, preventing concurrent access bugs at compile time.

- data races (simultaneously accessing the same memory with at least one write)
- dangling pointers (using memory after it's been deallocated)

#### Rust provides primitives for two main models: shared memory and message passing.
