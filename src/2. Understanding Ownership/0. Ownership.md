# Ownership

**Ownership** is Rust's way of managing memory without a garbage collector. Three rules:

1. Each value has **one owner**.
2. Only **one owner at a time**.
3. When the owner goes out of scope, the value is **dropped**.

## Stack vs Heap

| Stack | Heap |
|---|---|
| Fixed size, LIFO | Dynamic size |
| Fast | Slower (allocation + pointer chase) |
| Local variables | Box, Vec, String |

## Ownership & Moves

```rust
let s1 = String::from("hello");
let s2 = s1;          // s1 MOVED to s2
// println!("{s1}");  // ❌ error: s1 no longer valid
```

## Clone (deep copy)

```rust
let s1 = String::from("hello");
let s2 = s1.clone();   // ✅ deep copy
println!("{s1}, {s2}");
```

## Copy types (stack-only)

Scalars and tuples of scalars implement `Copy` — automatically copied.

```rust
let x = 5;
let y = x;            // x is Copy, still valid
println!("{x}, {y}");  // ✅
```

## References & Borrowing

A reference lets you use a value without taking ownership.

```rust
fn main() {
    let s1 = String::from("hello");
    let len = calculate_length(&s1);  // borrow, not move
    println!("'{s1}' length = {len}");  // s1 still usable
}

fn calculate_length(s: &String) -> usize {
    s.len()
}
```

### Rules of Borrowing

- You can have **any number of immutable references** (`&T`).
- OR **exactly one mutable reference** (`&mut T`).
- References must **always be valid** (no dangling pointers).

```rust
let mut s = String::from("hello");
let r1 = &s;          // ✅ multiple immutable OK
let r2 = &s;          // ✅
// let r3 = &mut s;   // ❌ can't borrow as mutable while immutable exists
```

## Slices

A slice is a **reference** to a contiguous sequence (no ownership).

```rust
let arr = [1, 2, 3, 4, 5];
let slice = &arr[1..3];         // [2, 3]

let s = String::from("hello world");
let hello = &s[0..5];           // "hello"
let world = &s[6..];            // "world"
```

> **Note:** `&str` is a string slice — the most common string type in function params.

## Non-Lexical Lifetimes (NLL)

Rust's borrow checker is smart — borrows end when the reference is **last used**, not when scope ends.

```rust
let mut x = 5;
let y = &mut x;
*y += 1;
// y is no longer used here — borrow ends (NLL)
let z = &x;   // ✅ allowed because y's borrow ended
println!("{z}");
```

This is **Non-Lexical Lifetimes** (NLL), stabilized in Rust 2018 edition.

## Lifetime Elision Rules (Detailed)

Three rules the compiler applies automatically to reduce annotation burden:

1. **Each** input reference gets its own lifetime parameter.
2. If there is **exactly one** input lifetime, it's assigned to all output references.
3. If `&self` or `&mut self` is an input, its lifetime is assigned to all output references.

```rust
// Rule 1: fn foo(x: &str, y: &str) → fn foo<'a, 'b>(x: &'a str, y: &'b str)
// Rule 2: fn foo(x: &str) -> &str → fn foo<'a>(x: &'a str) -> &'a str
// Rule 3: fn foo(&self, x: &str) -> &str → fn foo<'a, 'b>(&'a self, x: &'b str) -> &'a str
```

## Lifetime Variance

Variance describes how subtyping relationships propagate through type constructors.

- **Covariant** — `&'a T` is a subtype of `&'b T` if `'a` outlives `'b` (i.e., `'a: 'b`).
- **Invariant** — `&'a mut T` is invariant: you cannot substitute a longer lifetime.
- Most types are covariant in their lifetime parameters. `Cell<T>` and `&mut T` are invariant.
- Variance enables safe mutation through references.

> **Summary:** Ownership, borrowing, lifetimes, and variance form Rust's memory safety foundation. Ownership governs when values are dropped. Borrowing (references) allows temporary access without taking ownership. Lifetime parameters (`'a`) connect borrowed values to their sources, and the elision rules minimize the annotations you need to write. NLL makes the borrow checker more permissive by releasing borrows at their last use. Variance ensures that subtyping relationships (especially around lifetimes) remain sound, with `&mut T` and `Cell<T>` being invariant to prevent aliasing violations. Together, these concepts let Rust guarantee memory safety at compile time — no garbage collector, no dangling pointers, no data races.
