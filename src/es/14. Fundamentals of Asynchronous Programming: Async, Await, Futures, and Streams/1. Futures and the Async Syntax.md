# Futures y la Sintaxis Async

## El Trait `Future`

Un `Future` representa un valor que aún no está listo. El executor lo hace poll repetidamente:

```rust
use std::task::Poll;

trait SimpleFuture {
    type Output;
    fn poll(&mut self) -> Poll<Self::Output>;
}
// Poll::Ready(T)  → value is available
// Poll::Pending   → not ready yet, try again later
```

## `async` / `.await`

- `async fn` — el compilador transforma el cuerpo de la función en una **máquina de estado** que implementa `Future`.
- `.await` — cede el control al executor. La tarea se pausa hasta que el future esté listo.

```rust
async fn fetch_data() -> String {
    // This .await suspends the function without blocking the thread
    tokio::time::sleep(std::time::Duration::from_secs(1)).await;
    "Data received".to_string()
}

async fn main_async() {
    println!("Starting fetch...");
    let result = fetch_data().await; // pause here until ready
    println!("{result}");
}
```

## Lo que `async fn` Realmente Hace

```rust
// This:
async fn foo() -> u32 { 42 }

// Desugars into something like:
fn foo() -> impl Future<Output = u32> {
    async { 42 }
}
```

## Bloques `async`

También puedes crear futures en línea sin una función:

```rust
let future = async {
    println!("Hello from an async block!");
    42
};
let result = future.await;
```

---

## Puntos Clave

| Concepto | Significado |
|---------|---------|
| **`Future`** | Trait con `poll()` → devuelve `Ready(T)` o `Pending`. |
| **`async fn`** | Devuelve `impl Future<Output = T>`. No bloquea hasta que se hace await. |
| **`.await`** | Pausa el future actual, cede el control al executor. |
| **Máquina de estado** | El compilador genera un enum que rastrea el progreso a través de los puntos await. |
| **Necesita executor** | Una función async no hace nada hasta que un runtime como Tokio la hace poll. |
