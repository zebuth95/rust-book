# Aplicando Concurrencia con Async

Múltiples tareas asíncronas pueden ejecutarse **concurrentemente** (entrelazadas) en un solo hilo. El executor cambia entre ellas en los puntos `.await`.

## `tokio::join!` — Ejecutar Múltiples Futures, Esperar por Todos

```rust
use tokio::time::{sleep, Duration};

async fn task1() {
    println!("Task 1: Started");
    sleep(Duration::from_millis(100)).await;
    println!("Task 1: Finished");
}

async fn task2() {
    println!("Task 2: Started");
    sleep(Duration::from_millis(50)).await;
    println!("Task 2: Finished");
}

#[tokio::main]
async fn main() {
    // Both tasks run concurrently, finish in ~100ms (not 150ms)
    tokio::join!(task1(), task2());
}
```

## Bloques `async` — Futures en Línea

```rust
#[tokio::main]
async fn main() {
    let f1 = async {
        tokio::time::sleep(Duration::from_millis(100)).await;
        "first"
    };

    let f2 = async {
        tokio::time::sleep(Duration::from_millis(50)).await;
        "second"
    };

    let (a, b) = tokio::join!(f1, f2);
    println!("{a} and {b} done");
}
```

## Cómo Funciona la Concurrencia

```
Thread ──► Task A ──► .await (pending) ──► Task B ──► .await ──► Task A ready ──► ...
                │                            │
           waits for I/O                waits for I/O
```

- Ningún hilo se bloquea — el hilo **nunca está inactivo** mientras haya tareas listas.
- Esto es **cooperativo** — las tareas deben ceder el control con `.await` para que otras se ejecuten.

---

## Puntos Clave

| Concepto | Significado |
|---------|---------|
| **Concurrencia vs paralelismo** | Entrelazado (async) ≠ simultáneo (hilos en múltiples núcleos). |
| **Cooperativo** | Las tareas ceden voluntariamente en `.await`. Sin preferencia. |
| **`join!`** | Ejecuta futures concurrentemente, espera a que todos terminen. |
| **Monohilo** | Muchas tareas pueden ejecutarse en un solo hilo del SO — muy eficiente para I/O. |

#### `yield_now` — Cesión Cooperativa

```rust
tokio::task::yield_now().await; // voluntarily yields to the executor
```

Útil en bucles de larga duración con uso intensivo de CPU dentro de código async para evitar privar a otras tareas.

#### `tokio::task::JoinSet` — Gestión Dinámica de Tareas

```rust
use tokio::task::JoinSet;

let mut set = JoinSet::new();

for i in 0..5 {
    set.spawn(async move {
        tokio::time::sleep(std::time::Duration::from_millis(i * 10)).await;
        i
    });
}

// Process results as they complete
while let Some(result) = set.join_next().await {
    println!("completed: {:?}", result);
}
```

- `JoinSet` proporciona concurrencia estructurada — todas las tareas lanzadas se cancelan cuando `JoinSet` se descarta.
- `join_next().await` devuelve resultados a medida que las tareas se completan (no en orden de lanzamiento).

#### `tokio::task::LocalSet` — Futures No-`Send`

Usa `LocalSet` para ejecutar futures `!Send` en un solo hilo:

```rust
use tokio::task::LocalSet;

let local = LocalSet::new();
local.spawn_local(async {
    // Rc<RefCell<T>> is safe here — all on one thread
    let data = Rc::new(RefCell::new(0));
    *data.borrow_mut() += 1;
    println!("{}", data.borrow());
});

local.await;
```

> **Nota:** `spawn_local` requiere ejecutarse dentro de un `LocalSet` y desde el contexto de un `tokio::runtime::Handle`.
