# Separating Modules into Different Files

- This section further clarifies using the use keyword. It discusses how to handle cases where two items have the same name. You can use an alias with the as keyword to give one of the items a different name, for example use std::io::Result as IoResult;.

- The pub use combination is used for re-exporting. This makes an item public so that code outside of the module can use the item through your module's path.

- The * glob operator can be used to bring all public items defined in a path into the current scope.


    // In a module
        mod back_of_house {
            pub struct Breakfast {
                pub toast: String, // Public field
                seasonal_fruit: String, // Private field by default
        }
    }

In this example, toast can be accessed by other modules, but seasonal_fruit cannot.
