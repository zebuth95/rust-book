# Generic Data Types
Generics allow you to define functions, structs, enums, and methods that can work with
abstract types. This avoids code duplication and makes your code more flexible.
Rust's generics are zero-cost abstractions, meaning the compiler performs a process called
**monomorphization** (is the process ofd turning code into specific code by filling in the concrete types that are used when compile) at compile time,
creating a specific version of the code for each type used. This result in no runtime overhead.

**Runtime overhead refers to the extra resources (like time or memory) a computer uses for background operations or management that are not part of the core task being performed, ultimately slowing down the application or consuming more memory.** 

### Generic Function:

    fn largest<T: std::cmp::PartialOrd>(list: &[T]) -> &T {
        let mut largest = &list[0];
        for item in list {
            if item > largest {
                largest = item;
            }
        }
        largest
    }

The <T: std::cmp::PartialOrd> syntax is a trait bound that constrains T to types that can be compared.

**In Rust, T often represents a generic type parameter. This means T is a placeholder for any concrete type that will be specified later when the code using the generic construct is instantiated. This allows for writing flexible and reusable code that can work with various types without needing to duplicate the logic for each specific type.**

### Generic Struct:

    struct Point<T> {
        x: T,
        y: T,
    }

    let integer_point = Point { x: 5, y: 10 };
    let float_point = Point { x: 1.0, y: 4.0 };
