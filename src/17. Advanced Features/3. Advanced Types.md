# Advanced Types

## Newtype Pattern

```rust
struct Meters(f64);
struct Kilometers(f64);

impl Meters {
    fn to_kilometers(&self) -> Kilometers { Kilometers(self.0 / 1000.0) }
}
```

## Type Aliases

```rust
type Kilometers = i32;
type Thunk = Box<dyn Fn() + Send + 'static>;
```

## Never Type (`!`)

A function that never returns:

```rust
fn forever() -> ! {
    loop { /* ... */ }
}
```

## Dynamically Sized Types (`?Sized`)

```rust
fn generic<T: ?Sized>(t: &T) {
    // T can be unsized (e.g., str, [u8])
}
```

> **Key point:** `!` is the never type — used in `continue`, `panic!`, infinite loops.

#### Const Generics

Generic parameters that are **compile-time constants** (not types). Stabilized incrementally from Rust 1.51.

```rust
struct Array<T, const N: usize> {
    data: [T; N],
}

impl<T: std::fmt::Debug, const N: usize> Array<T, N> {
    fn print(&self) {
        println!("{:?}", self.data);
    }
}

let small: Array<i32, 3> = Array { data: [1, 2, 3] };
let large: Array<i32, 100> = Array { data: [0; 100] };
```

Const generics can be used with:
- Array sizes (`[T; N]`)
- Any `usize`, `u8`, `i32`, `bool`, `char` or other integer types
- Expression evaluation at compile time

```rust
fn dot_product<const N: usize>(a: [f64; N], b: [f64; N]) -> f64 {
    let mut sum = 0.0;
    for i in 0..N { sum += a[i] * b[i]; }
    sum
}
```

> **Key point:** Const generics eliminate runtime size checks and enable type-safe fixed-size data structures.
