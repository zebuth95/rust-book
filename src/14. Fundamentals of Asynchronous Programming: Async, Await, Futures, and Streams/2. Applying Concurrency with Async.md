# Applying Concurrency with Async

Multiple async tasks can run **concurrently** (interleaved) on a single thread. The executor switches between them at `.await` points.

## `tokio::join!` — Run Multiple Futures, Wait for All

```rust
use tokio::time::{sleep, Duration};

async fn task1() {
    println!("Task 1: Started");
    sleep(Duration::from_millis(100)).await;
    println!("Task 1: Finished");
}

async fn task2() {
    println!("Task 2: Started");
    sleep(Duration::from_millis(50)).await;
    println!("Task 2: Finished");
}

#[tokio::main]
async fn main() {
    // Both tasks run concurrently, finish in ~100ms (not 150ms)
    tokio::join!(task1(), task2());
}
```

## `async` Blocks — Inline Futures

```rust
#[tokio::main]
async fn main() {
    let f1 = async {
        tokio::time::sleep(Duration::from_millis(100)).await;
        "first"
    };

    let f2 = async {
        tokio::time::sleep(Duration::from_millis(50)).await;
        "second"
    };

    let (a, b) = tokio::join!(f1, f2);
    println!("{a} and {b} done");
}
```

## How Concurrency Works

```
Thread ──► Task A ──► .await (pending) ──► Task B ──► .await ──► Task A ready ──► ...
                │                            │
           waits for I/O                waits for I/O
```

- No threads are blocked — the thread is **never idle** as long as tasks are ready.
- This is **cooperative** — tasks must yield with `.await` for others to run.

---

## Key Points

| Concept | Meaning |
|---------|---------|
| **Concurrency vs parallelism** | Interleaved (async) ≠ simultaneous (threads on multiple cores). |
| **Cooperative** | Tasks voluntarily yield at `.await`. No preemption. |
| **`join!`** | Runs futures concurrently, waits for all to complete. |
| **Single-threaded** | Many tasks can run on one OS thread — very efficient for I/O. |

#### `yield_now` — Cooperative Yielding

```rust
tokio::task::yield_now().await; // voluntarily yields to the executor
```

Useful in long-running CPU-bound loops within async code to prevent starving other tasks.

#### `tokio::task::JoinSet` — Dynamic Task Management

```rust
use tokio::task::JoinSet;

let mut set = JoinSet::new();

for i in 0..5 {
    set.spawn(async move {
        tokio::time::sleep(std::time::Duration::from_millis(i * 10)).await;
        i
    });
}

// Process results as they complete
while let Some(result) = set.join_next().await {
    println!("completed: {:?}", result);
}
```

- `JoinSet` provides structured concurrency — all spawned tasks are cancelled when `JoinSet` is dropped.
- `join_next().await` returns results as tasks complete (not in spawn order).

#### `tokio::task::LocalSet` — Non-`Send` Futures

Use `LocalSet` to run `!Send` futures on a single thread:

```rust
use tokio::task::LocalSet;

let local = LocalSet::new();
local.spawn_local(async {
    // Rc<RefCell<T>> is safe here — all on one thread
    let data = Rc::new(RefCell::new(0));
    *data.borrow_mut() += 1;
    println!("{}", data.borrow());
});

local.await;
```

> **Note:** `spawn_local` requires running inside a `LocalSet` and from within a `tokio::runtime::Handle` context.
