# Traits: Definiendo Comportamiento Compartido

Traits = la versión de Rust de las **interfaces**. Definen un conjunto de métodos que los tipos pueden implementar.

## Definiendo un Trait

```rust
pub trait Summary {
    fn summarize(&self) -> String;                           // no default
    fn summarize_author(&self) -> String {
        format!("(Read more from {})", self.summarize())      // default impl
    }
}
```

## Implementando un Trait

```rust
pub struct NewsArticle {
    pub headline: String,
    pub author: String,
    pub location: String,
}

impl Summary for NewsArticle {
    fn summarize(&self) -> String {
        format!("{}, by {} ({})", self.headline, self.author, self.location)
    }
}

pub struct Tweet {
    pub username: String,
    pub content: String,
}

impl Summary for Tweet {
    fn summarize(&self) -> String {
        format!("{}: {}", self.username, self.content)
    }
}
```

## Traits como Parámetros

```rust
// `impl Trait` syntax — accepts any type implementing `Summary`
pub fn notify(item: &impl Summary) {
    println!("Breaking news! {}", item.summarize());
}
```

### Sintaxis de Bound de Trait (equivalente)

```rust
pub fn notify<T: Summary>(item: &T) {
    println!("Breaking news! {}", item.summarize());
}
```

### Múltiples Bounds de Trait

```rust
// Must implement BOTH Summary AND Display
pub fn notify(item: &(impl Summary + Display)) {}

// Same with generic syntax
pub fn notify<T: Summary + Display>(item: &T) {}

// Cleaner with `where` for complex bounds
pub fn notify<T, U>(item: &T) -> String
where
    T: Summary + Display,
    U: Clone + Debug,
{}
```

## Retornando `impl Trait`

```rust
fn returns_summarizable() -> impl Summary {
    Tweet {
        username: String::from("horse_ebooks"),
        content: String::from("people"),
    }
}
```

> **Nota:** Solo se puede retornar **un tipo concreto** con `impl Trait`. Para múltiples tipos, usa `Box<dyn Trait>`.

---

## Puntos Clave

| Concepto | Significado |
|---------|---------|
| **Implementación por defecto** | Los métodos pueden tener cuerpos por defecto. Los implementadores pueden sobrescribirlos. |
| **Bound de Trait** | `T: TraitName` — restringe un genérico a tipos que implementan el trait. |
| **`impl Trait`** | Abreviatura de `T: Trait` en parámetros de función. |
| **Sintaxis `+`** | Combina múltiples bounds: `T: TraitA + TraitB`. |
| **Cláusula `where`** | Sintaxis más limpia para bounds de trait complejos. |
