# How to Write Tests

Tests are defined as regular functions annotated with the #[test] attribute. The cargo test command finds and executes these functions. Tests often rely on assertion macros:

- assert!: Ensures a boolean expression is true.
- assert_eq!: Ensures two values are equal (==).
- assert_ne!: Ensures two values are not equal (!=).

### These macros are powered by the traits PartialEq and Debug (for error reporting).

## Custom Failure Messages:
All assertion macros can take optional arguments to format a custom failure message, which is helpful for debugging complex test failures.

## Checking for Panics:
The #[should_panic] attribute is used to assert that an operation should cause a panic!. You can also specify an expected substring to ensure the panic message is correct.


## Basic Assertion:

    #[test]
    fn check_addition() {
        assert_eq!(2 + 2, 4, "2 + 2 should always equal 4");
    }

## Checking for a Panic:

    #[test]
    #[should_panic(expected = "Guess value must be less than or equal to 100")]
    fn greater_than_100() {
        // Assume Guess::new panics if value > 100
        Guess::new(200);
    }
