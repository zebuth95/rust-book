# The match Control Flow Construct
Rust has an extremely powerful control flow construct called **match** that allows you
to compare a value against a series of patterns and then exclude code based on which
pattern matches.

    enum Coin {
        Penny,
        Nickel,
        Dime,
        Quarter,
    }
    
    fn value_in_cents(coin: Coin) -> u8 {
        match coin {
            Coin::Penny => 1,
            Coin::Nickel => 5,
            Coin::Dime => 10,
            Coin::Quarter => 25,
        }
    }

Let’s imagine that a friend is trying to collect all 50 state quarters.
While we sort our loose change by coin type, we’ll also call out the name of the
state associated with each quarter so that if it’s one our friend doesn’t have,
they can add it to their collection.


    #[derive(Debug)] // so we can inspect the state in a minute
    enum UsState {
        Alabama,
        Alaska,
        // --snip--
    }
    
    enum Coin {
        Penny,
        Nickel,
        Dime,
        Quarter(UsState),
    }
    
    fn value_in_cents(coin: Coin) -> u8 {
        match coin {
            Coin::Penny => 1,
            Coin::Nickel => 5,
            Coin::Dime => 10,
            Coin::Quarter(state) => {
                println!("State quarter from {state:?}!");
                25
            }
        }
    }
    State quarter from Alaska!

In the **match** expression for this code, we add a variable called **state**
to the pattern that matches values of the variant **Coin::Quarter**.

If we were to call **value_in_cents(Coin::Quarter(UsState::Alaska))**,
coin would be **Coin::Quarter(UsState::Alaska)**.


## Matching with Option<T>
In the previous section, we wanted to get the inner **T** value out of the **Som** case
when using **Option<T>**; we can also handle **Option<T>** using **match**.

    fn plus_one(x: Option<i32>) -> Option<i32> {
        match x {
            None => None,
            Some(i) => Some(i + 1),
        }
    }

    let five = Some(5);
    let six = plus_one(five);
    let none = plus_one(None);

first execution of **plus_one**, When we call **plus_one(five)**, the variable x in the 
body will have the value **Some(5)**. We then compare that against each match arm:

- None => None,
- Some(i) => Some(i + 1),

Does **Some(5)** match **Some(i)**? it does! We have the same variant.


## Matches Are Exhaustive
**match** must cover all possibilities.

    fn plus_one(x: Option<i32>) -> Option<i32> {
        match x {
            Some(i) => Some(i + 1),
        }
    }

It not contains a handle for **None** causing a bug.

## Catch-All Patterns and the _ Placeholder
Using enums, we can also take special actions for a few particular values.

    let dice_roll = 9;
    match dice_roll {
        3 => add_fancy_hat(),
        7 => remove_fancy_hat(),
        _ => reroll(),
    }

    fn add_fancy_hat() {}
    fn remove_fancy_hat() {}
    fn reroll() {}

Finally, we’ll change the rules of the game one more time so that nothing else happens on your turn if you roll anything other than a 3 or a 7.

    let dice_roll = 9;
    match dice_roll {
        3 => add_fancy_hat(),
        7 => remove_fancy_hat(),
        _ => (),
    }

    fn add_fancy_hat() {}
    fn remove_fancy_hat() {}
